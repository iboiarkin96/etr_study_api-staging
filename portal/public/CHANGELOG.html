Changelog

Notable changes to the public surface of the ETR Study API. Each entry covers the contract consumers depend on: HTTP endpoints, request/response shapes, stable error codes, security defaults, SLO targets. Internal refactors that do not affect any of the above are out of scope and live in the SA-internal release log.

1.0.0 — 2026-06-06

Initial public release.

  • Added. /api/v1/user — composite-key user resource (POST · GET · PATCH · PUT) with the standard Idempotency-Key contract on writes.
  • Added. Error envelope with stable code field; COMMON_* + USER_* code families published in the error catalogue.
  • Added. Defensive defaults documented in the security model (header-key auth, CORS off, body cap, header redaction, rate limit).
  • Added. Availability and latency SLO targets, with rationale, in the SLO & error budget page.
  • Added. Diátaxis-shaped portal — Tutorials · How-to · Reference · Explanation — with the OpenAPI explorer mounted at reference / api.

Upgrade policy

  • Additive changes (new endpoint, new optional field, new response field, new error code) ship as a minor bump and do not require client work.
  • Breaking changes (renamed/removed field, tighter validation, repurposed code) land behind a new major prefix (/api/v2) with at least one overlap release.
  • See explanation: API versioning for the full additive-vs-breaking table.
