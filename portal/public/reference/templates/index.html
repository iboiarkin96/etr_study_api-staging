Reference page templates

Canonical shapes for new Reference docs. Pick the closest template, copy its skeleton, and replace the placeholder content. Templates describe form (mandatory sections, ordering, required pills) — the procedural how of writing them lives in the SA-internal handbook.

Templates

  • API spec page — for documenting one endpoint, request/response shapes, examples.
  • Error catalogue page — for the public-facing list of stable codes.
  • Environment-variable page — for the consumer-relevant configuration surface.

When to author by hand vs. auto-generate

Auto-generate first. If the source of truth (OpenAPI, env-profile file, error registry) can feed the page, write a generator and treat the rendered HTML as a derivative. Hand-authored Reference is justified only when the source of truth is the prose itself — for example, a page describing the shape of the error envelope, not the codes.
