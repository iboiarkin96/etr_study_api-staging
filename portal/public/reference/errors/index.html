Error catalogue

Every non-2xx response uses one envelope. Switch on code, never on detail. See handle errors for the recovery recipe.

The tables below mirror the code entries declared in openapi-baseline.json and are reviewed against it on every release. When in doubt, the OpenAPI baseline wins — this page is a human index over it, not a parallel source of truth.

COMMON_* — transport, auth, idempotency

These errors can be returned by any endpoint. They reflect framing, authentication, idempotency, and infrastructure conditions — not business rules.

CodeStatusMeaning
COMMON_000 · COMMON_VALIDATION_ERROR422Generic validation fallback when no specific (field, type) rule matches.
COMMON_400 · IDEMPOTENCY_KEY_REQUIRED400Missing required Idempotency-Key header on a write operation.
COMMON_401 · SECURITY_AUTH_REQUIRED401Missing or invalid X-API-Key.
COMMON_409 · IDEMPOTENCY_KEY_REUSED_WITH_DIFFERENT_PAYLOAD409Same idempotency key seen with a different request body within the TTL.
COMMON_413 · SECURITY_REQUEST_BODY_TOO_LARGE413Request body exceeds the configured maximum size.
COMMON_429 · SECURITY_RATE_LIMIT_EXCEEDED429Caller exceeded the per-key rate limit; honor Retry-After.

USER_* — user resource business and validation

Returned by /api/v1/user endpoints. Business codes (USER_101+, USER_404) indicate domain conflict; validation codes (USER_001–USER_0xx) appear under 422's errors[] array.

CodeStatusMeaning
USER_101 · USER_CREATE_ALREADY_EXISTS409Composite key (system_user_id, system_uuid) already in use.
USER_102 · USER_PATCH_BODY_EMPTY400PATCH body has no fields to update.
USER_404 · USER_NOT_FOUND404No user with the given composite key.

The full validation matrix (PATCH and PUT codes too) is rendered into the OpenAPI explorer per-endpoint examples and is the source of truth.

Code stability

  • Stable identifier: the code field. Once published, a code's meaning never changes; deprecated codes are retired with a release note.
  • Mutable: the detail message — phrasing may change for clarity. Do not regex-match it.
  • Numeric prefix (USER_001, COMMON_409) is only an organisational hint; rely on the symbolic name (USER_CREATE_FULL_NAME_REQUIRED) when switching in client code.
