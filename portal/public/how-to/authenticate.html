How-to

Authenticate with the API

Send the API key in the X-API-Key request header on every protected endpoint under /api/v1. The key is a long-lived shared secret — no token-exchange flow, no per-request signature.

The recipe

curl "$BASE_URL/api/v1/user/$SYSTEM_UUID/$SYSTEM_USER_ID" \
      -H "X-API-Key: $API_KEY"

Verify

Confirm the header is reaching the server: omit the key once and check the failure shape, then restore it and check the success path.

# Expect 401 SECURITY_AUTH_REQUIRED
curl -s -o /dev/null -w '%{http_code}\n' "$BASE_URL/api/v1/user/$SYSTEM_UUID/$SYSTEM_USER_ID"

# Expect 200 with a JSON body
curl -s -w '\n%{http_code}\n' "$BASE_URL/api/v1/user/$SYSTEM_UUID/$SYSTEM_USER_ID" \
      -H "X-API-Key: $API_KEY"

If the first call returns anything other than 401, your gateway is bypassing the API key check; if the second is 401, the header name or value is wrong (most common: Authorization instead of X-API-Key).

What goes wrong

StatusCodeMeaningWhat you do
401 SECURITY_AUTH_REQUIRED (COMMON_401) Header missing or value not recognised. Re-check the header spelling (X-API-Key, not Authorization) and the key value. Do not retry blindly.
413 SECURITY_REQUEST_BODY_TOO_LARGE (COMMON_413) Body exceeds the configured maximum. Trim the payload; the limit is published in the OpenAPI baseline.
429 SECURITY_RATE_LIMIT_EXCEEDED (COMMON_429) Too many requests from the caller. Back off; respect the Retry-After hint when present.

Operational notes

  • Transport: always HTTPS in production. The local-dev base URL is plain HTTP for convenience; do not commit or paste plain-HTTP keys anywhere a network can see them.
  • Storage: keep the key in a secret manager or environment variable, never in the repo, never in logs. The server already redacts X-API-Key headers in structured logs.
  • Rotation: rotate proactively if a key may have leaked. There is no built-in expiry — operators rotate explicitly.
  • One key per integration: if you have several environments (dev / staging / prod), use distinct keys so logs and rate-limit metrics are attributable.
