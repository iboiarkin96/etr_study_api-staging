Runbook SEV-2

Portal docs build is failing

Page builds error out and the published portals are stuck on the last good commit. Reader-facing impact: stale docs. Author impact: cannot ship updates.

Owner: Portal team On-call: @ivan MTTR target: 30 min

Escalation

L1 — author L2 — portal-team L3 — platform

Detect

Triggered by CI status on main turning red on the portal-build job, or by an author reporting "my page didn't publish."

Diagnose

  1. Open the latest failed run

    Find the failing step and scroll to the first ERROR line.

    gh run view --log-failed

  2. Classify the error

    HTML validation? Sidebar JSON parse? Search-index build? Each has a different fix path.

Mitigate

  1. Revert if a single commit is responsible

    git revert <sha> && git push origin main

    Verify: the next build runs green and the changed pages serve from the published portal.

    Rollback: if the revert breaks a different page, git revert the revert and proceed to the next step.

  2. Patch forward if revert is too disruptive

    Apply the smallest targeted fix on a hotfix branch and merge directly to main.

Post-incident

Write a one-page note: what broke, how we found out, why it slipped past CI locally. Add the missed check to the PR review checklist.
