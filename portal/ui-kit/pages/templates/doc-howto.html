How-to

Add a new internal section

Wire a new top-level section under /internal/<section>/ end-to-end: filesystem entry, sidebar node, search index, breadcrumbs.

Prerequisites

  • Local checkout up to date with main.
  • Authorial rights on services/portal/internal/.
  • You have read RFC 0003 · Documentation authoring.

1 · Create the directory and index page

Pick a single-word slug. Add a directory and an index.html based on the landing-section template.

mkdir -p services/portal/internal/yoursection
cp services/frontend/portal/assets_v2/ui-kit/templates/landing-section.html \
      services/portal/internal/yoursection/index.html

Verify: services/portal/internal/yoursection/index.html exists and opens without 404 in the browser.

2 · Add the section to the sidebar tree

Open the mock JSON and append the new node. The sidebar component re-renders automatically — no JS change needed.

  { "id": "uml", "label": "UML", "href": "/internal/uml/", "children": [] },
      + { "id": "yoursection", "label": "Your section", "href": "/internal/yoursection/", "children": [] }

Verify: reload an internal page; the sidebar shows "Your section" in alphabetical order.

3 · Register in the search index

Append an entry to search-index-internal.json so the topbar search finds your landing.

{
"title": "Your section",
"breadcrumb": "Internal · Your section",
"href": "/internal/yoursection/",
"snippet": "Short summary that will appear under the title."
      }

4 · Add child pages

For every doc, copy the matching template: doc-article, doc-howto, doc-tutorial, doc-reference, or doc-explanation. Breadcrumbs auto-fill from the URL — you do not need to hand-write them.

Troubleshooting

SymptomLikely causeFix
Sidebar doesn't show the new nodeJSON parse errorRun jq . nav-tree-uikit.json — fix the offending comma
Breadcrumbs render "Yoursection"No human label registeredAdd a mapping in SEGMENT_LABELS in breadcrumbs.js
Search returns nothingIndex not yet rebuiltRestart the dev server or trigger the indexer manually
