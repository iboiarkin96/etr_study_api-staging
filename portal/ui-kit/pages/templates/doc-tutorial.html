Tutorial

Build your first UI-kit page

In about twenty minutes you'll ship a doc page that uses every key piece of the v2 kit: topbar, sidebar, breadcrumbs, TOC, code blocks, and footer history.

⏱ 20 min 📚 Beginner 🧰 HTML only

1Copy the template

Start with the article template — it's the most flexible and fits 80% of docs. 

cp services/frontend/portal/assets_v2/ui-kit/templates/doc-article.html \
        services/portal/internal/handbook/my-first-page.html

✓ Check your progress

The new file is present, and opening it in the browser shows the kit chrome with placeholder text.

2Edit content

Replace {{ page.title }} with your title, drop your prose between <article> tags, and write h2/h3 headings with stable id attributes — the TOC populates from those. 

<h2 id="intro">Introduction</h2>
        <p>Your prose goes here.</p>

3Add it to the sidebar

Append a node to nav-tree-uikit.json under the matching section.

4Preview and ship

Open the page in the browser. Confirm: breadcrumbs auto-built, TOC tracks headings, dark mode works, sidebar highlights the active item. If all checks pass — open a PR.
