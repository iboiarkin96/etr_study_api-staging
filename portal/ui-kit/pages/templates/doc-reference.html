API · v1

Sessions

Reference for the /api/v1/sessions resource — fields, endpoints, error codes.

stable v1

Resource

A Session represents a single study sitting — a contiguous period in which a user reviews one or more cards. Sessions are created when the user opens a deck and closed when they leave the app or pass an idle threshold.

Fields

Field Type Required Constraints Description
id uuid ● server-assigned Unique identifier.
user_id uuid ● Owner; foreign key to users.id.
deck_id uuid ● Deck under review during this session.
started_at timestamp ● ISO-8601, UTC Server clock; set on create.
ended_at timestamp ○ ISO-8601, UTC Null while the session is open.
duration_ms int ○ ≥ 0 Derived; only set when ended_at is.
cards_reviewed int ● ≥ 0 Count of reviews rows attached.
tags string[] ○ ≤ 16 chars each Free-form labels.

Endpoints

GET /api/v1/sessions/{id} Fetch a single session by ID

Fetch a single session. Returns the full session object or 404 if not found.

Request

bash$ curl -sS https://api.study-app.example/v1/sessions/$ID \
              -H "Authorization: Bearer $TOKEN" | jq .

Response 200

json{
"id": "0e5b…",
"user_id": "0a32…",
"deck_id": "8f01…",
"started_at": "2026-05-11T18:24:00Z",
"ended_at": "2026-05-11T18:42:11Z",
"duration_ms": 1091000,
"cards_reviewed": 24,
"tags": ["evening", "core-vocab"]
              }
POST /api/v1/sessions Open a new study session

Open a new session. Server fills id, started_at, and a default cards_reviewed: 0.

Request

bash$ curl -sS https://api.study-app.example/v1/sessions \
-H "Authorization: Bearer $TOKEN" \
-H "Content-Type: application/json" \
              -d '{"deck_id":"8f01…"}'

Response 201

json{
"id": "1a7c…",
"user_id": "0a32…",
"deck_id": "8f01…",
"started_at": "2026-05-14T09:00:00Z",
"ended_at": null,
"duration_ms": null,
"cards_reviewed": 0,
"tags": []
              }
PATCH /api/v1/sessions/{id} Close a session and stamp ended_at

Close a session and stamp ended_at. The server computes duration_ms.

Request

bash$ curl -sS -X PATCH https://api.study-app.example/v1/sessions/$ID \
-H "Authorization: Bearer $TOKEN" \
-H "Content-Type: application/json" \
              -d '{"cards_reviewed":24,"tags":["evening"]}'

Response 200

json{
"id": "0e5b…",
"ended_at": "2026-05-14T09:42:11Z",
"duration_ms": 2531000,
"cards_reviewed": 24
              }

Errors

StatusCodeMeaning
400 session_already_open User has an active session — close it first.
404 session_not_found No row matches the supplied id (or it isn't yours).
409 session_already_closed Cannot patch a closed session.
422 invalid_deck Deck does not exist or is archived.
429 rate_limited Too many session opens — see Retry-After header.
500 internal_error Unexpected server error. Retry with exponential backoff.
