Prose

Typographic rhythm for docs prose. Provides defaults for raw HTML (h1–h4, p, lists, blockquotes, figures) inside .docs-shell > main and .docs-prose. All selectors are wrapped in :where() so UI Kit component classes trivially win — prose offers defaults, never overrides.

Scope

Prose applies in two places:

  • .docs-shell > main — every internal page that uses the shell layout (the vast majority).
  • .docs-prose — opt-in container for prose blocks outside the shell (e.g., embedded sections, modals, marketing surfaces).

Specificity is (0,0,0) on every rule via :where(). Any component class — .docs-card__name, .docs-chip-row, .pa-eyebrow — wins automatically. Don't worry about overrides when authoring a new component.

Headings

Four levels with shrinking margin, line-height, and weight. h2 and h3 carry scroll-margin-top for anchor links under the sticky topbar.

H1 — page title (--fs-900)

H2 — section title (--fs-800)

Paragraphs & lists

Body paragraphs use --fs-400 with --lh-base. Margins below paragraphs and lists are --space-4; list items get --space-1 between siblings.

Block elements

Blockquote

Blockquote with a --rail-w left rail in --accent, muted text, and a soft background. Use sparingly — for pull-quotes and emphasised guidance, not as a generic callout (use .docs-callout for that).

Page head helpers

.page-head wraps the title + lede of every page; it sits inside <main class="container"> directly above the article body and renders the title, the floating top-nav anchor, and the muted lede paragraph.

  • .page-head — bottom rule and spacing.
  • .page-head h1 — top margin reset.
  • .page-head p — muted lede, max-width clamped to --layout-prose-max.
  • .docs-prose__lede / .lede — standalone lede paragraph anywhere inside prose.
  • .kit-section — UI Kit pages use this to add extra top-margin between top-level sections; > h2 resets its own top-margin so it doesn't double-stack.

Tokens used

  • Type scale — --fs-200, --fs-400, --fs-500, --fs-600, --fs-700, --fs-800, --fs-900.
  • Line-height — --lh-tight, --lh-snug, --lh-base.
  • Weight — --fw-semibold, --fw-bold.
  • Spacing — --space-1 … --space-7.
  • Colour — --text, --text-strong, --muted, --muted-soft, --line, --line-soft, --bg-mute, --accent.
  • Layout — --layout-topbar (scroll-margin), --layout-prose-max (lede clamp), --rail-w (blockquote rail).

Why :where()

Without :where(), a rule like .docs-shell > main h2 has specificity (0,1,2) and would silently override a component class like .docs-card__title at (0,1,0). That caused real alignment drift across pages mixing prose with kit components. Wrapping the scope selector in :where() zeroes its contribution, so prose becomes a true default layer: any class wins by default.
