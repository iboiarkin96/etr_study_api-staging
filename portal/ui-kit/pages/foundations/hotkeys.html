Foundations

Hotkeys

Keyboard first; mouse is a fallback. The kit honours a small, fixed map — three groups, no surprises, no chord conflicts with the OS.

Contract

  1. Single-key shortcuts only fire when no input is focused. Type a slash inside a textarea and you get a slash, not the search palette.
  2. Chord shortcuts use g as the leader (Gmail-style). g s = "go to sidebar", g h = "go home".
  3. Modifier shortcuts use ⌘/Ctrl only where they extend an existing OS metaphor (⌘+K = palette, ⌘+/ = help).
  4. No conflicts with browser defaults. We never rebind ⌘+T, ⌘+W, ⌘+R, etc.

The map

Navigation

KeysAction
/Focus topbar search
⌘+KOpen command palette (when wired)
g→hGo to portal home
g→sFocus sidebar (opens drawer on mobile)
g→tFocus right-rail TOC
EscClose any open overlay (drawer, modal, toc-fab panel)

View & theme

KeysAction

[Collapse / expand sidebar
]Collapse / expand TOC right rail
zToggle reading mode (hides chrome — sidebar + TOC + topbar fades to minimal)

Help

KeysAction
?Open this hotkeys map as a modal cheat-sheet
⌘+/Same as ?; the OS-native modifier version

Implementation notes

  • The runtime is one module: ui-kit/components/hotkeys.js. It owns the document-level keydown listener, the chord buffer (1.2 s timeout), and the editable-target guard so single-key shortcuts never fire while typing.
  • Each action reaches into the component that owns it via DOM (theme-toggle button, sidebar collapse caret, TOC toggle, drawer trigger). The module never duplicates state — pressing t clicks the visible flashlight button, so the visible affordance stays the single source of truth.
  • mountHotkeys() is idempotent and runs last in boot() so all targets exist when bindings register.
  • A binding without a visible affordance is a bug. Every shortcut shows up in the ? cheat-sheet, which is built lazily on first press.

Status

This page is the contract, and as of 2026-05-16 the runtime matches it. The full map is live across both portals via ui-kit/components/hotkeys.js: the g-leader chord runtime, the ? cheat-sheet modal, single-key bindings (t / [ / ] / z), modifier shortcuts (⌘+K, ⌘+/), and the global Esc dismiss. Try ? now to see the live cheat-sheet.
