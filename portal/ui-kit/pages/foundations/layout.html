Layout

.docs-shell is the page grid every internal page uses. Three columns on desktop (sidebar · main · TOC), one column under 1025 px. Defined in ui-kit/components/layout.css and applied via <body class="docs-shell">.

Grid template

The shell is a CSS grid with grid-template-columns: var(--layout-sidebar) minmax(0, 1fr) auto. The middle column has min-width: 0 so wide tables and code blocks can scroll without blowing out the layout.

Anatomy

  • .docs-shell — the grid container, capped at --layout-shell-max and centred via margin-inline: auto.
  • .docs-shell > .topbar — header that spans all columns; its background runs viewport-edge to viewport-edge while inner padding aligns with the shell cap.
  • .docs-shell > .docs-sidebar — first column, row 2.
  • .docs-shell > main — second column, row 2; the article body. Padding: --space-5 --space-6 --space-7 on desktop.
  • .docs-shell > .docs-toc — third column, row 2; the on-this-page nav with a right margin of --space-5.

Modifiers

.is-sidebar-collapsed

Toggled by the sidebar's collapse button. Swaps the first column to --layout-sidebar-collapsed. The transition runs at 200ms --ease-out (and is disabled under prefers-reduced-motion: reduce).

.docs-shell--full

Landing/marketing pages opt out of the sidebar/TOC. Removes the shell cap so the hero can fill the viewport, and lets <main> span all three grid columns. Inner padding clamps to --layout-content-max * 1.4.

Responsive tiers

Per the mobile-contract canon — uniform typography across viewports, only spacing/columns shift:

  • ≥ 1025 px (desktop) — three columns; sidebar pinned; TOC visible.
  • 761 – 1024 px (tablet) — single column, sidebar & TOC hidden, main padding drops to --space-4 --space-4 --space-6. Mobile drawer takes over navigation.
  • ≤ 760 px (mobile) — single column; main padding drops further to --space-3 --space-3 --space-5.

Pitfalls

  • Don't cap the article wrapper itself. Prose wrappers (<article> / .docs-prose) used to clamp at --layout-prose-max here; that left a wide right gap on every prose template. The cap is now applied per element (.page-head p, .kit-section > p) so wide elements (tables, code, cards) fill the column.
  • Don't put a fixed-width child in main without min-width:0. The middle column already carries min-width: 0; nested grid items should follow the same rule or you'll lose horizontal scroll on wide tables.
  • Topbar background isn't part of <main>. If you need an edge-to-edge band inside main, wrap it in a width: 100vw; margin-inline: calc(50% - 50vw) escape — same trick the topbar uses.

Tokens used

  • --layout-sidebar, --layout-sidebar-collapsed — sidebar column width.
  • --layout-topbar — header row height.
  • --layout-shell-max — outer cap of the grid.
  • --layout-content-max — used by .docs-shell--full for landing-page padding.
  • --space-3 … --space-7 — main padding tiers.
  • --ease-out — sidebar collapse transition.
