Inline diagram

Bordered figure wrapping an inline SVG diagram, with a caption underneath. Comes with a typed library of SVG helper classes (boxes / lines / arrows / text) that read design tokens, so the diagram re-themes automatically between light and dark.

Example

Anatomy

  • .pa-figure — bordered box, --bg-elev background, hidden overflow so wide SVGs clip cleanly.
  • .pa-figure svg — block, centered, max-width:100% so the diagram is responsive.
  • .pa-figure__caption — small italic caption below the SVG.
  • SVG primitives:
    • .pa-svg-box — neutral rectangle (stroke --line, fill --bg-elev). Modifiers: --accent, --good, --bad.
    • .pa-svg-line — neutral 1.5px line; --dash dashed.
    • .pa-svg-arrow — same colour as a line, marker-friendly.
    • .pa-svg-text — body text inside the SVG; --mono, --muted, --accent, --good, --bad tonal modifiers.

Markup skeleton

<figure class="pa-figure" aria-label="…">
  <svg viewBox="0 0 W H" role="img" aria-label="…">
    <defs>
      <marker id="arrow" …><path d="M0 0 L10 5 L0 10 z" style="fill: var(--muted)"/></marker>
    </defs>
    <rect class="pa-svg-box pa-svg-box--accent" …/>
    <text class="pa-svg-text pa-svg-text--accent" …>Label</text>
    <path class="pa-svg-line" d="…" marker-end="url(#arrow)"/>
  </svg>
  <figcaption class="pa-figure__caption">…</figcaption>
      </figure>

A11y

  • Add aria-label on both the <figure> and the inner <svg role="img"> with a short description of what the diagram conveys.
  • Long-form text goes in <figcaption> — screen readers read both.
  • SVG text uses real <text> nodes (not paths), so screen readers can read the labels too.

Naming note

Class names use the legacy .pa-figure / .pa-svg-* prefix; neutral rename to .inline-diagram / .diagram-* queued.
