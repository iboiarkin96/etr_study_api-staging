Template annotator

Drop-in template-anatomy overlay. Adds a pill toggle to the topbar; one click reveals numbered callouts that point at every well-known block on the page — what it is, and which UI Kit component to use to fill it.

When to use

  • Template documentation. Anatomy callouts replace screenshot-and-arrow diagrams that rot.
  • Design hand-off. Reviewers and new authors see "this region is the lede, fill it with .lede inside .docs-prose" right on the live page.
  • Component discovery. Toggle once on any template — see every reusable block on it numbered with the exact UI Kit class to use.

Live demo

Every page in Templates has the annotator wired in. Open any template, click the Enable annotations pill in the topbar — left of the theme toggle — and you'll see numbered callouts for every block present on that page.

Setup — two lines

The annotator is fully opt-in. No template needs custom markup, IDs or data-ann attributes — blocks are matched by CSS selector from the registry. To enable it on a page, link the CSS in <head>:

<link rel="stylesheet"
      href="…/services/frontend/portal/assets_v2/ui-kit/components/template-annotator.css">

…and load the module before </body>:

<script type="module"
      src="…/services/frontend/portal/assets_v2/ui-kit/components/template-annotator.js"></script>

The script auto-injects the toggle pill into .topbar__actions (or floats it top-right if no topbar exists) and the SVG overlay onto <body>. State is not persisted — every page load starts with annotations off.

Why opt-in (not in entry.css)

Annotations are an authoring/learning aid for the ~15 kit template pages — they're not useful on the other ~700+ portal pages. Bundling template-annotator.css + .js into the runtime entry would ship ~3 KB on every page for a feature only the kit templates use. The two-line opt-in keeps the runtime lean; the cost is remembering to add the link when authoring a new template. The block registry is the only file you need to touch when adding a new annotatable block.

Behaviour

Toggle

The pill sits in the topbar and reads Enable annotations when off, Disable annotations when on. ON state fills the pill with the accent colour and pulses the leading dot.

Numbered callouts

  • One right-side rail (220 px wide) holds every box, sorted top-to-bottom by element position.
  • Numbering is page-local — restarts from 1 on each page, regardless of the block's global ID in the registry.
  • Boxes never overflow above the page edge or under the topbar: the first one is pinned below header.topbar's bottom edge (falls back to y ≥ 14 px when no topbar is present).
  • L-shaped dashed leader line with a small badge in its middle segment connects each box to its element.

Expand / collapse

  • By default every box is compact: number · short label · chevron.
  • Click to expand into description + monospace component tag.
  • When one box is open, its leader line is solid + bolder, lifted above all others; all other lines and boxes dim so the active relationship reads clearly.

Auto-detection rules

  • Elements with display:none or zero getBoundingClientRect() are skipped (e.g. toc-fab on desktop).
  • Elements with position:fixed are skipped (e.g. rocket) — they don't scroll with the page, so a leader tied to in-flow boxes would jump.
  • Each registry entry maps to the first match of its selector — we annotate one representative paragraph, one card, one pill, not every occurrence.

Block registry — 45 entries

All annotatable blocks live in BLOCK_REGISTRY inside template-annotator.js. Each entry is one row of:

{
block: 'h1', selector: 'main h1, article.docs-prose h1, .page-head h1',
what: 'Article title (h1)',
desc: 'Main page heading. Exactly one h1 per page.',
comp: 'docs-prose h1'
      }

Adding a new block

  1. Open services/frontend/portal/assets_v2/ui-kit/components/template-annotator.js.
  2. Append a new entry to BLOCK_REGISTRY with a unique block ID, a selector that matches the new element, and the three text fields (what, desc, comp).
  3. Reload any template that contains the new element — the box appears automatically. No template-level changes required.

Composite selectors (a, b, c) work — the first match wins. Use them when the same block can live under several wrappers (e.g. nav.topbar__breadcrumbs, .docs-breadcrumbs).

A11y

  • Toggle is a real <button> with aria-pressed reflecting the on/off state.
  • SVG overlay carries aria-hidden="true" — annotations are a visual aid, not assistive content.
  • Annotation boxes are real DOM (clickable to expand) and announce their label/description via plain text.
  • Mode is OFF on every page load — never a surprise overlay for screen-reader users.

Tokens used

  • --accent — leader lines, badges, pill ON state, dot indicator
  • --bg, --card, --line, --muted, --text — surfaces and text
  • --layout-sidebar, --layout-topbar — grid columns + button offset
  • --shadow-md — floating toggle fallback shadow
  • --radius-sm, --font-mono — pill / comp-tag styling
