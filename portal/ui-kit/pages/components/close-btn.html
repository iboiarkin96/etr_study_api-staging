Close button

Generic dismiss affordance — a small square icon-button used by modals, drawers, toasts, banners, and any other transient surface. Ghost by default so it sits inside coloured panels without competing; picks up --bg-mute on hover.

Sizes

sm 28×28 · md (default) 32×32 · lg 40×40

Tones

default · --ghost (inherits text colour) · --danger

SVG glyph

For sharper rendering at large sizes use an inline SVG instead of the × character. The component scales the glyph via width: 1em.

Inside a modal

Add data-modal-close so the modal controller picks the button up automatically — no extra wiring required.

Dialog title

Header carries a .docs-close-btn[data-modal-close]; the controller wires it automatically.

Markup

<!-- Default -->
<button type="button" class="docs-close-btn" aria-label="Close">×</button>

<!-- Size + tone -->
<button type="button" class="docs-close-btn docs-close-btn--sm docs-close-btn--danger"
aria-label="Dismiss">×</button>

<!-- Modal-wired (no JS bind needed) -->
<button type="button" class="docs-close-btn" data-modal-close
      aria-label="Close" data-tooltip="Close (Esc)">×</button>

A11y

  • Always provide aria-label — the × glyph alone is not an accessible name.
  • Use type="button" so the button never accidentally submits an enclosing form.
  • Pair with a data-tooltip="Close (Esc)" for keyboard users when the affordance is visually subtle.
  • Inside a modal: the controller already handles Esc and backdrop-click; the button is the third dismissal path for mouse-only users.

Tokens used

  • --muted, --text, --bg-mute, --line
  • --radius-sm
  • --accent (focus ring), --danger (danger variant)
  • --ease-out
