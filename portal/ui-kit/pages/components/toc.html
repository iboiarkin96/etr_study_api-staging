Table of contents

Right-rail "On this page" navigation for long-form documentation. Two levels mirror the article's <h2> / <h3> structure; scroll-spy flashes the active section; a one-click toggle hands the column's width back to <main> when the reader wants a wider read. On viewports ≤ 1024 px the rail is hidden and the toc-fab takes over as a bottom-sheet opener.

Example

A typical article TOC: three top-level h2 entries plus two h3 sub-points nested under the currently-active section. The active item carries the accent left-bar; h3 children are indented and use the smaller type ramp.

Anatomy

Markup is one <aside class="docs-toc"> at the end of the page. mountToc() finds it, then:

  1. Builds the list from headings — collects every h2[id] / h3[id] inside <main>. h3s nest under the most recent h2; orphan h3s (before any h2) gain .docs-toc__item--h3. h4+ are deliberately skipped to keep the rail scannable.
  2. Wires the collapse toggle — wraps the title in .docs-toc__header, injects a <button.docs-toc__toggle> with a chevron SVG and tooltip "Collapse / Expand the on-this-page rail". State persists in localStorage["docs-toc-collapsed"]. Collapsed: the aside drops to a fixed right-edge tab and the body gains data-toc-collapsed="true" so layout.css reclaims the column for <main>.
  3. Wires scroll-spy — one IntersectionObserver watches every heading with rootMargin: "0px 0px -65% 0px". On change, the runtime picks the heading whose top sits inside the upper 35 % of the viewport (with a mid-viewport fallback when the user is near the bottom of the page); the matching .docs-toc__item gets --active.
  4. Bails out gracefully — fewer than 2 headings? The aside is hidden (toc.hidden = true). No main? The mount returns without touching the DOM.

Markup

Author the markup without the toggle button — mountToc() injects it on boot. Hierarchy is DOM-driven: nest a second <ol class="docs-toc__list"> inside an <li> to add h3 children.

<aside class="docs-toc" aria-label="On this page">
<p class="docs-toc__title">On this page</p>
<ol class="docs-toc__list">
<li class="docs-toc__item">
<a class="docs-toc__link" href="#intro">Introduction</a>
</li>
<li class="docs-toc__item">
<a class="docs-toc__link" href="#why">Why it matters</a>
<ol class="docs-toc__list">
<li class="docs-toc__item"><a class="docs-toc__link" href="#why-a">Reader cost</a></li>
<li class="docs-toc__item"><a class="docs-toc__link" href="#why-b">Author risk</a></li>
</ol>
</li>
</ol>
</aside>

<!-- Flat-list alternative when nesting isn't practical -->
        <li class="docs-toc__item docs-toc__item--h3">…</li>

States

  • Default link — colour --muted, 2 px transparent left-bar, line-height 1.4.
  • Hover — colour --text; left-bar stays transparent (it's reserved for the active state).
  • Focus-visible — 2 px --accent outline, 2 px offset.
  • Active — 2 px --accent left-bar, colour --accent, weight --fw-medium. Driven by scroll-spy: the heading inside the top 35 % of the viewport wins; ties broken by document order.
  • h3 link — same anatomy, indented to --space-5, type drops to --fs-100, colour --muted-soft.
  • Aside collapsed ([data-collapsed]) — leaves the grid flow, becomes a position: fixed tab on the right edge with rounded left corners and the toggle's chevron rotated to point ←.
  • Body collapsed (body[data-toc-collapsed="true"]) — grid column 3 shrinks to 0 so <main> reclaims the width. Persists in localStorage["docs-toc-collapsed"].
  • Mobile (≤ 1024 px) — display: none on the rail; toc-fab takes over.

A11y

  • Wrap the rail in <aside aria-label="On this page"> — it's a complementary landmark, not navigation.
  • Use an <ol>: the document outline is ordered, the TOC mirrors that order.
  • Every heading the TOC references must carry a stable id that matches the link's href fragment.
  • The toggle is a real <button type="button"> with aria-label="Toggle table of contents" and aria-expanded that flips with the collapsed state.
  • The flash-on-active transition is bounded by the global prefers-reduced-motion override in tokens/_base.css; no per-component opt-out needed.
  • Tab order inside the rail follows document order. The toggle is the last focusable element so keyboard readers reach the content before discovering the collapse affordance.
  • On mobile the rail is removed from the accessibility tree (display: none); discoverability moves to toc-fab, whose button carries its own aria-controls.

Tokens used

  • --card (collapsed tab surface), --line, --line-soft
  • --muted, --muted-soft, --text, --text-strong
  • --accent (active left-bar, active text, focus outline)
  • --radius-sm, --radius-md (collapsed tab corners)
  • --shadow-sm (collapsed tab lift)
  • --space-1 … --space-6
  • --fs-100 (h3 link + title eyebrow), --fs-200 (base link size)
  • --fw-medium (active), --fw-semibold (title)
  • --ease-out
  • --layout-topbar (sticky top offset and fixed-tab anchor)
