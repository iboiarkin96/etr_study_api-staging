Practice article

Page-template component. The 150 role-practice pages (sa · architect · dev · manager · qa · sre — 25 practices each) share a fixed long-form skeleton. The skeleton is composed from 12 atomic kit components plus a small residual of practice-only primitives. This page documents the umbrella: which atoms compose the article, in what order, with what tone, and what stays in practice-article.css itself.

Live exemplars

  • internal/roles/sa/practices/01-diataxis-discipline — full canonical practice (group A, accent tone).
  • internal/roles/qa/practices/07-mutation-testing — same skeleton, group C (teal).
  • internal/roles/sre/practices/19-error-budgets — group D (warn).

Anatomy — top-to-bottom

  1. Eyebrow (.pa-eyebrow) — number badge + group label + before/after scorebar. Practice-only.
  2. Title + lede (.pa-title + .pa-lead) — large h1 plus a muted intro paragraph. Practice-only.
  3. TL;DR callout — one-sentence answer at the top.
  4. Inline diagram — SVG diagram with theme-aware tokens.
  5. Compare grid — «as-is» vs «to-be» columns.
  6. State strip — finite-state visualisation of the practice's lifecycle.
  7. Quality checklist — pass/fail rubric.
  8. Pitfalls table (.pa-pitfalls) — common mistakes; first column wide-and-bold. Practice-only.
  9. Rollout timeline — phases over time.
  10. Worked example — concrete walked-through case.
  11. Anatomy grid — parts breakdown of a structure.
  12. Industry cards — how Netflix / Stripe / Spotify do this.
  13. Deep-dive callout — «if you want more, read X».
  14. References grid — closing two-column reading list.
  15. Section card — neutral body-section wrapper used between rich primitives.
  16. Page-history footer — canonical footer.docs-history.

Not every practice uses every block. The canonical pattern (per the practice-page review-loop feedback) drops scorebar, page-history table, and standalone «See also» blocks; keeps the rich .pa-* blocks and inline-diagram SVGs with <defs><marker> arrows.

What stays in practice-article.css

After the decomposition, the bundle file shrunk from 771 → ~118 lines. The residual is everything tied to «one practice page» as a whole, not reusable elsewhere:

  • .pa-eyebrow + __num / __group — number badge + group label row.
  • .pa-scorebar + __asis / __arr / __tobe / __hint — before/after score pill.
  • .pa-title, .pa-lead — practice-page typography (separate from the prose ladder).
  • .pa-pitfalls — pitfalls table tightening.
  • .pa--gA … .pa--gE — 5-group accent overrides driving --pa-accent.

Twelve former .pa-* blocks moved out as atomic kit components — each with its own showcase linked above.

5-group tone taxonomy

Every practice belongs to one of five groups. Group accent drives --pa-accent, which colours the eyebrow badge, the inline-diagram strokes, the state-strip rail, and the TL;DR callout rail.

  • .pa--gA → --accent (blue) — discipline / foundations
  • .pa--gB → #8b5cf6 (violet) — architecture / strategy
  • .pa--gC → --teal — observability / testing
  • .pa--gD → --warn — risk / reliability
  • .pa--gE → --rose — failure-modes / postmortems

Dark theme overrides each token so the accents stay readable on dark surfaces (defined inside practice-article.css).

Adding a new practice

  1. Create services/portal/internal/roles/<role>/practices/<NN-slug>/index.html.
  2. Copy the skeleton from a sibling practice in the same role (the structure is identical across the 150 pages).
  3. Set the wrapper class to the appropriate group (.pa--gA … .pa--gE).
  4. Drop the blocks you need in the order listed above. Every block has a showcase you can copy markup from.
  5. Add the practice to the role's radar (services/portal/internal/roles/<role>/radar/index.html) so it shows up in the role hub.
  6. Run .venv/bin/python scripts/validate_docs_design.py on the new page before committing.

Why the .pa- prefix stays

Class names use the legacy .pa-* prefix on purpose: 150 consumer pages reference these classes. Renaming would touch ~3 000 occurrences for cosmetic gain. The 12 atoms that moved out got neutral names (tldr-callout, state-strip, etc.) because they're used outside practice pages too. The five practice-only blocks above stay .pa-* — they are practice-page chrome.
