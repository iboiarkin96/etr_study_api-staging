SA lifecycle

Boxed lifecycle region — short prose lede on top, a 4-up grid of state cards below (state name + git transition + one-sentence hint). Used to explain a page's freshness / approval / supersession states in one self-contained block.

Example

Page lifecycle is driven by git — each state corresponds to a specific commit type touching the page's data-last-verified attribute.

  • Fresh data-last-verified bumped within SLA

    No banner, no action. The page is within its type-specific staleness window.

  • Stale-soon SLA expiry < 30 days away

    Banner appears; owner pinged weekly until re-verified or deprecated.

  • Stale last-verified date past SLA

    Red banner; owner on the hook to re-verify or deprecate within two weeks.

  • Deprecated moved to /deprecated/ tree

    Reader sees a tombstone with a link to the replacement. Removed from search index.

Anatomy

  • .lifecycle-box — bordered box, --bg-elev background, large padding.
  • .lifecycle-box__lede — 64ch max-width opening paragraph; inline <code> gets a muted background tint.
  • .lifecycle-box__legend — 4-column grid (2 cols at ≤1024 px, 1 col at ≤760 px).
  • Each <li> — accent-tinted left border + muted accent-wash background, three stacked rows: state · git · hint.
  • .lifecycle-box__state — mono uppercase, --muted-soft.
  • .lifecycle-box__git — mono, --text-strong, breaks on long paths.
  • .lifecycle-box__hint — small body copy, --muted.

Markup

<div class="lifecycle-box">
  <p class="lifecycle-box__lede">…</p>
  <ul class="lifecycle-box__legend">
    <li>
      <span class="lifecycle-box__state">Fresh</span>
      <span class="lifecycle-box__git">data-last-verified bumped within SLA</span>
      <p class="lifecycle-box__hint">No banner, no action.</p>
    </li>
    <!-- repeat for the other states -->
  </ul>
      </div>

Tokens used

  • Layout: --space-3 / -4 / -5, --radius-lg, --radius-sm, --radius-xs.
  • Surface: --bg-elev, --bg-mute, --line.
  • Colour: --accent (left rail tint), --text-strong, --muted, --muted-soft.
  • Typography: --fs-100 / -200, --font-mono, --lh-snug, --lh-base.
