Entity card

Backstage-style catalog descriptor that doubles as the page header on every services/<name>/index.html. Replaces the legacy page-head + h2 + svc-meta trio with a single tone-tinted block whose halo, accent rail, chip row, fields, and footer line up with the rest of the kit.

Anatomy

The card is split into three stacked regions inside one panel:

  • Head — chip row (.entity-card__pills) with lifecycle + facets, mono <h1> title with a leading · in the tone colour, and a one-paragraph lede (max 72 ch).
  • Fields — a <dl> rendered as a 2-column grid (160-220 px / 1 fr) with dashed hairlines between rows; each <dt> shows an uppercase mono key with a tone-coloured bullet dot.
  • Foot — mono row with Maintainer, Reviewed (timestamp), and Spec (descriptor + RFC link).

Two decorative layers carry the tone: a corner ::before halo in the top-right and a 2-px ::after accent rail along the top edge. Both are purely visual — set data-tone="…" on the section and they pick up the tone automatically.

Tones

Each catalog entry picks a tone that matches its lifecycle or its kind: stable services typically use info or accent, experimental ones warning, design-system entries success, deprecated / runbook-only entries danger.

Markup

The component is structural-only — no runtime JS. Pick a tone via data-tone, fill in chips / title / lede / fields / foot.

Content guidelines

  • Pills: first chip is always the lifecycle (stable / experimental / deprecated); follow with stack / facet chips. Keep the row to ~4 chips.
  • Lede: one paragraph, ≤ 72 ch wide, written so it stands on its own — no "see below". Inline links are fine; underline tracks the tone.
  • Fields: pick from a stable vocabulary (Owner, Lifecycle, Provides, Depends on, Repo, Code reference, Links, Source, Showcase). Use <span class="entity-card__sep" aria-hidden="true">·</span> between inline items in a single <dd>.
  • Foot: always three slots — Maintainer · Reviewed · Spec. <time datetime> for the timestamp.

A11y

  • Use <section> with aria-labelledby pointing at the title element so the card reads as a landmark.
  • The leading · in the title is decorative (::before, not in the DOM) — screen readers won't announce it.
  • The corner halo and top rail are pointer-events: none with aria-hidden implied (CSS-only). They never trap focus or steal clicks.
  • The inline · separator inside <dd> uses aria-hidden="true" so it's not read out between items.
  • Anchor underlines are tone-tinted but always at ≥ 3:1 contrast against the card background; hover state pumps to the full tone for clarity.

Mobile

Below 760 px the fields grid collapses to a single column. <dt> stacks above <dd> (no hairline between key and value), and the foot row tightens its gap. Halo + accent rail keep their proportions.

Tokens used

  • --svc-tone (local) ← --info / --success / --warn / --danger / --accent
  • --bg-elev, --line, --line-soft
  • --text, --text-strong, --muted, --muted-soft
  • --space-1 … --space-6, --radius-lg
  • --font-mono, --fs-300, --fs-400, --fs-700, --fs-900
  • --ease-out (link-underline transition)
