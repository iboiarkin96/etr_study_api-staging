Concept layers

A nested matryoshka diagram for layered concepts where each outer layer wraps the layer below and adds one more thing. The canonical case is SLI ⊂ SLO ⊂ SLA — the layout itself carries the claim that one concept is built on top of the others.

Example

SLA

Service-Level Agreement

= SLO + money penalty

A contract with a paying customer.

SLO

Service-Level Objective

= SLI ≥ target, over a window

The target value of the SLI over a window. An internal promise.

SLI

Service-Level Indicator

= raw measurement

A measurement we can read from the system. Examples:

  • Share of HTTP requests with status 2xx or 3xx.
  • p95 response time of GET /conspectuses/due.

Tones

The data-tone attribute swaps the per-layer accent (border, background tint, rule colour, examples bullet).

  • data-tone="rose" — --rose (outermost, contract-like).
  • data-tone="accent" — --accent (middle, target-like). Default if omitted.
  • data-tone="info" — --info (innermost, raw measurement).
  • data-tone="teal" — --teal (additional, e.g. four-layer concepts).

Layers are real DOM nesting — the visual inset comes from the parent's padding plus a top accent bar (::before). There is no fixed maximum depth, but two or three layers is the readable sweet spot.

Responsive behaviour

  • ≥761 px — full padding (--space-5), titles at --fs-500.
  • ≤760 px — padding shrinks to --space-4, titles drop to --fs-400, head gap tightens so the inner-most layer still has breathing room.

A11y

  • The outer container takes an aria-label summarising the nesting (e.g. “SLI inside SLO inside SLA”).
  • Titles are real headings (<h3>) under the page's <h2> — keep the order intact.
  • The __rule span is rendered monospaced; wrap the inner text in <code> when it is a formula so it survives copy-paste.
  • Component-scoped selectors (.docs-concept-layers .docs-concept-layers__title) intentionally raise specificity so prose.css defaults (h3 → fs-700; p → bottom margin) do not bleed through.

Tokens used

  • --accent, --rose, --info, --teal — per-tone borders, tints, rule colour.
  • --bg-elev, --bg-mute, --line — surface and code-chip background.
  • --text-strong, --text, --muted — title, examples, note copy.
  • --radius-lg, --radius-xs, --radius-full — layer corner, code chip, examples bullet.
  • --fs-100 … --fs-500; --fw-semibold.
  • --font-mono — rule text.
  • --space-1 … --space-7.
