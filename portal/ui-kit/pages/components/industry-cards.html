Industry cards

Case-studies grid. Each card carries a company tag, a short «what they did» paragraph, an italic «lesson» callout, and a source link. Useful for «how do Stripe/Spotify/Netflix actually do this» context boxes — typically 3-6 cards under a section heading, optionally followed by a single italic «pattern» summary line.

Example

Stripe

Treats every public API surface as a versioned contract; generates seven SDKs from one OpenAPI source on every release.

Lesson: the schema, not the code, is the source of truth.

stripe.com/docs/api

Spotify

Backstage models every running service as a typed entity; the catalog is the spine of the developer portal.

Lesson: «what runs in production» belongs in a typed registry, not in a wiki.

backstage.io

Netflix

Chaos Monkey injects failures continuously in production so resilience stays observed, not assumed.

Lesson: if you don't break it on purpose, it'll break on its own at 3 am.

netflixtechblog.com

Across all three, the pattern is the same: turn an aspiration into a daily mechanism that runs without anyone deciding to run it.

Anatomy

  • .pa-industry — auto-fit grid, minimum 260 px per card.
  • .pa-industry__card — bordered tile with a 3 px tone-coloured left bar (::before).
  • .pa-industry__company — mono uppercase tag, tone-coloured.
  • .pa-industry__what — semibold prose, what they actually did.
  • .pa-industry__lesson — tinted block, the takeaway. <strong> inside is tone-coloured.
  • .pa-industry__source — pinned to the bottom; the link is tone-coloured.
  • .pa-industry-pattern — optional italic summary line that follows the grid.

Markup

</div>
      <p class="pa-industry-pattern">…optional cross-card pattern line…</p>
