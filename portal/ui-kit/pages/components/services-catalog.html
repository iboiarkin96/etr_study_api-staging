Services catalog

Page-template component. Renders the service-catalog hub at services/portal/internal/services/index.html — one tile per Backstage service (.svc-tile) with a per-tile spine sub-nav, the entity meta-list, a filter-chips bar, and the architecture overview band with a hand-drawn C4 L1 diagram. The tile auto-renders from catalog-info.yaml via scripts/render_service_descriptors.py.

Live exemplars

  • internal/services/index.html — the catalog hub (full template; 4 service tiles).
  • internal/services/api/index.html — a single service detail page (uses .entity-card + .svc-spine).

Anatomy — what the page contains

  1. Hero band — page hero with a filter-chips row and an inline info trigger inside the h1 explaining the catalog policy.
  2. Architecture overview — a section with a hand-drawn C4 L1 diagram inside a <figure> wrapper and a collapsible PUML source toggle.
  3. Service tiles (.svc-tile) — one tile per service. Each tile contains:
    • .svc-tile__head — icon + service name (anchored link).
    • .svc-tile__lede — one-sentence description.
    • .svc-meta — entity meta-list (Owner, Lifecycle, Provides, …).
    • .svc-spine — chip row of canonical pages (Overview, Architecture, API reference, Runbooks, On-call). Missing pages render as .svc-spine__missing tooltip-explained placeholders.
  4. Spine structure band — 6-step tile row for the catalog hero showing «what every service owes»; collapses to 2-col at modal widths.

Per-service tones

Each service tile gets a tone via a modifier class, which drives --svc-tone:

  • .svc-tile--api → var(--info)
  • .svc-tile--portal → var(--accent)
  • .svc-tile--monitoring → var(--warn)
  • .svc-tile--uikit → var(--success)

The tone colors the hover halo, the focus ring, the spine chip accents, and the service-name link. Add a new modifier when registering a new service.

Renderer

Service tiles are generated, not hand-written: scripts/render_service_descriptors.py reads each services/portal/internal/services/<name>/catalog-info.yaml and updates three marker-bounded regions in internal/services/index.html (entity-card, catalog-grid, catalog-tickers) plus the nav tree. Run make catalog-render after editing a service descriptor.

Adding a new service

  1. Create services/portal/internal/services/<name>/catalog-info.yaml following the Backstage descriptor format (see service-descriptor template).
  2. Create services/portal/internal/services/<name>/index.html — the service detail page (use the api service as a live exemplar).
  3. Add a .svc-tile--<name> tone modifier in services-catalog.css if the four built-in tones don't fit.
  4. Run make catalog-render to regenerate the catalog hub + nav tree.
