Modal

Dialog with backdrop, focus trap, and ESC-to-close. Built on the native <dialog> element where supported.

Example · static

Discard changes?

You have unsaved changes in this section. Discarding will revert to the last saved state.

Variants

  • Default — centered, 440 px wide.
  • Wide — modifier .docs-modal--wide, 720 px.
  • Form modal — body contains a <form>; foot has Submit / Cancel.
  • Confirm — primary action uses danger colour for destructive intents.

Markup

<div class="docs-modal" role="dialog" aria-modal="true"
aria-labelledby="modal-title" aria-hidden="true">
<div class="docs-modal__backdrop"></div>
<div class="docs-modal__panel">
<div class="docs-modal__head">
<h2 id="modal-title">Discard changes?</h2>
<!-- Use the generic close-button component; `data-modal-close`
lets the modal controller auto-bind it. -->
<button class="docs-close-btn" type="button" data-modal-close
aria-label="Close" data-tooltip="Close (Esc)">×</button>
</div>
<div class="docs-modal__body">…</div>
<div class="docs-modal__foot">
<button class="docs-btn docs-btn--secondary" data-modal-close>Keep editing</button>
<button class="docs-btn docs-btn--primary">Discard</button>
</div>
</div>
      </div>

States

  • Closed — hidden attribute set, no DOM cost beyond the element.
  • Opening — 160 ms fade-in (backdrop) + 200 ms scale-up (panel) with --ease-out.
  • Open — focus trapped inside, body scroll locked.
  • Closing — symmetric exit; focus returned to trigger.

A11y

  • Use role="dialog" and aria-modal="true".
  • Provide aria-labelledby pointing at the title or aria-label directly.
  • Trap focus inside; ESC closes; click on backdrop closes (unless modal is destructive — then require explicit dismiss).
  • Return focus to the trigger on close.
  • Prefer native <dialog> with showModal() — it handles focus & ESC for free.

Tokens used

  • --bg-elev, --line, --line-soft
  • --shadow-lg, --radius-lg
  • --space-4, --space-5
  • --ease-out
