Deep-dive callout

Opt-in box for deeper-than-baseline content the reader can skip on first pass. A tone-coloured top edge sets the title apart; the inner h3 reads in the tone colour, h4s read as sub-headings, prose / lists / code get tighter margins so the block reads as a self-contained aside.

Example

Why the migration is two passes, not one

Doing it in one pass would let writes and reads disagree for the duration of the migration. The two-pass shape pairs writes to both stores during the cut-over so the read path can switch with a single flag flip.

Edge case: re-reads inside a transaction

Inside a single transaction, the new read path can hit the old store if the row hasn't been backfilled yet. The shim transparently falls back; CI has a Schemathesis test that exercises this path.

def get_card(card_uuid):
    row = new_store.get(card_uuid)
    if row is None:
        row = legacy_store.get(card_uuid)  # transparent fallback
          return row

Anatomy

  • .pa-deepdive — bordered region with a 3 px tone-coloured top edge.
  • Inner <h3> — section title in the tone colour, semibold.
  • Inner <h4> — sub-heading; --text-strong.
  • Inner <p>, <ul>, <ol>, <pre> — tightened margins so the block reads compact.

Markup

<aside class="pa-deepdive" aria-label="Deep dive">
  <h3>…</h3>
  <p>…</p>
  <h4>…</h4>
  <p>…</p>
  <pre><code>…</code></pre>
      </aside>

A11y

  • Use <aside> as the wrapping element so screen readers announce it as supplementary content.
  • Add an aria-label describing what kind of deep-dive it is («Why this approach», «Edge cases», «Math», …).

Naming note

Class names use the legacy .pa-deepdive prefix; neutral rename to .deepdive-callout queued.
