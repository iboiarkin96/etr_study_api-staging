Status timeline

Horizontal lifecycle chain for ADR, RFC, and Explanation documents. Tracks Proposed → Accepted → Active → Superseded → Deprecated with an accent glow on the current state, a metadata strip below, and a rose-tinted successor link-card when the document has been superseded.

Markup

Use the data-component="status-timeline" mount point and let the JS build the chain. Or write the BEM HTML directly for static rendering (as in the demos above).

<!-- JS-mounted (recommended for templates) -->
<div data-component="status-timeline"
data-current="accepted"
data-decided="2026-05-10"
data-supersedes="ADR 0017"
data-supersedes-href="/adr/0017"
data-author-id="16fc8b78537109162984a2fdbef6e142">
</div>

<!-- For superseded documents, add: -->
<div data-component="status-timeline"
data-current="superseded"
data-decided="2024-11-03"
data-superseded-by="ADR 0035"
data-superseded-by-href="/adr/0035"
data-superseded-by-title="New CI Pipeline Architecture"
data-author-id="...">
</div>

<!-- Static HTML (for showcase / no-JS) — set --st-progress to the index of
the current node (0..4) so the accent fill stops at the right dot -->
<nav class="docs-status-timeline" aria-label="Document status">
<ol class="docs-status-timeline__chain" role="list" aria-label="Document lifecycle" style="--st-progress: 1">

<span class="docs-status-timeline__dot" aria-hidden="true"></span>

States

Node role Dot Label
Current Accent fill + glow ring (box-shadow) --accent, --fw-semibold
Past Tinted accent fill + white checkmark overlay --muted-soft
Future Transparent + --line border --muted, 45% opacity
Connector: a single straight track sits behind all dots — dashed --line by default, with an accent-tinted solid overlay filling from the first dot up to the current dot (driven by --st-progress). When data-current="superseded", a rose-tinted successor link-card appears below the metadata strip.

A11y

  • The chain is an <ol role="list"> with aria-label="Document lifecycle" — conveying order without implying interactivity.
  • The current node carries aria-current="step"; all other nodes use aria-current="false".
  • Decorative dots and SVG icons use aria-hidden="true" — labels carry the semantic content.
  • The successor link-card is a real <a> element with visible focus ring (inherits the global :focus-visible treatment from button.css).
  • The metadata strip is a <dl> with explicit <dt> / <dd> pairs inside <div> wrappers — valid and screen-reader friendly.
  • The whole component is wrapped in <nav aria-label="Document status">, making it a distinct navigation landmark that screen readers can jump to.
  • Reduced-motion: no transition animations fire when prefers-reduced-motion: reduce is set.

Tokens used

  • --accent — current dot fill, glow, label colour, solid connector
  • --rose — successor card border, icon, heading, ID badge
  • --line — future dot border, dashed connector, meta strip border
  • --bg-mute — meta strip surface
  • --bg-elev — successor card background base
  • --muted, --muted-soft — future and past labels, meta keys
  • --fg — meta values
  • --radius-md, --radius-full — meta strip, dots, successor card
  • --shadow-sm — successor card hover lift
  • --fs-100 — meta keys, labels; --fs-200 — meta values, successor body; --fs-300 — successor title
  • --fw-medium, --fw-semibold, --fw-regular
  • --space-2 … --space-6
  • --font-mono — successor ADR ID
