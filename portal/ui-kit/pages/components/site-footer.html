Site footer

Global portal footer auto-mounted by mountSiteFooter() in runtime/{internal,public}/entry.js. Pages do not need to declare any markup — if no [data-component="site-footer"] slot exists, one is created inside .docs-shell after <main>. Configuration cascades from built-in defaults → window.__PORTAL_FOOTER__ → inline JSON → remote JSON via data-footer-src.

The real page-level footer at the bottom of this page shows what every portal page renders by default. The framed demos below illustrate configuration overrides and variants in isolation.

Default (4 columns + brand + legal)

What every portal page renders when nothing is configured. The same configuration mounts as the full-bleed footer at the bottom of this page; the framed copy below lets you see brand block, columns, and legal row side-by-side without scrolling.

Inline configuration

Pages override the footer by emitting an inline JSON config before the slot. Top-level keys (brand, columns, legal) are deep-merged onto defaults; arrays replace.

<script type="application/json" data-footer-config>
{ "brand": { "title": "…" }, "columns": [ … ], "legal": { … } }
</script>
              <footer data-component="site-footer" data-cols="3"></footer>

Page-level overrides

  • <html data-footer="off"> — disables auto-mount entirely.
  • window.__PORTAL_FOOTER__ = { … } set before entry.js loads — full config override.
  • data-footer-src="/path/to/footer.json" on the slot — async fetch, defaults render first then swap.
  • data-variant="compact|minimal" on the slot.
  • data-cols="2|3|4|5" on the slot to force column count.

A11y

  • Renders into a <footer> with role="contentinfo" and a default aria-label="Site footer".
  • External links carry rel="noopener noreferrer" + target="_blank" and a visual ↗ suffix.
  • Social icons have aria-label; SVG glyphs use aria-hidden="true".
  • Reduced-motion users get no hover transitions.

Tokens used

  • --bg-elev, --bg-mute — surface blend
  • --line, --line-soft — rules
  • --muted, --muted-soft — link/meta colors
  • --accent — brand glyph + link badges
  • --space-5/6/7 — vertical rhythm
  • --fs-200/300/500, --font-mono
