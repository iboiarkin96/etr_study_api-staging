Dependency lanes

Categorised list of pill-shaped dependencies. Each lane is a labelled row (Framework / Persistence / Schema / …) of small chips; each chip pairs a tone-tinted SVG glyph with a mono name and an optional hint. Used in the Depends on field of every entity-card on /services/<name>/index.html.

Anatomy

  • Wrapper — .dep-lanes with data-tone="info|success|warning|danger|accent". Sets the --dep-tone custom property that every child reads.
  • Row — .dep-lanes__row, a 2-column grid (140 px label / 1 fr chips) with a dashed hairline between rows in the tone.
  • Label — .dep-lanes__label, uppercase mono with a leading tone-coloured bullet dot.
  • Chip — .dep-lanes__chip, pill with .dep-lanes__icon + .dep-lanes__name. Optional trailing .dep-lanes__hint for inline annotations (next, optional, version, etc.).
  • Status — set data-status="planned" on a chip to flip it into a dashed warn-tinted variant. Default current.
  • Footnote — optional .dep-lanes__more below the lanes for the canonical "see the full graph" link.

Tones

Pick the tone of the host entity card so the lanes' hairlines and chip borders carry the same identity. info for stable services, success for design-system entries, warning for experimental, danger for deprecated, accent as default.

Status — current vs planned

data-status="planned" on a chip flips it into a dashed, warn-tinted variant — independent of the host tone. Use it for dependencies that are documented in the architecture but not yet wired up in production code.

Content guidelines

  • Lane labels are roles, not implementations. "Persistence" not "ORM & DB". Reader should be able to scan labels and understand the service's shape without reading the chips.
  • One chip per dependency. Don't pack "PostgreSQL 16 + pgBouncer" into one chip — give them their own lanes ("Database", "Connection pool") or separate chips in the same lane.
  • Hints are short and uppercase. next, optional, v3, :8000/metrics. Anything longer goes into the body of dependencies.html.
  • Glyphs reuse the kit's stroke vocabulary. 24 × 24 viewBox, stroke-width="2", stroke-linecap="round". Pick a glyph that signals the role, not the product logo.
  • Always pair with .dep-lanes__more if a fuller dependencies.html exists — the lanes are a summary, not the architecture document.
