Glossary list

A-Z glossary surface — a <dl> of term ↔ definition rows, grouped under per-letter sections, with an optional jump bar at the top. Used by the onboarding glossary and any vocabulary-reference page. Three pieces compose the surface: the A-Z jump bar, the per-letter group, and the term list.

Example

ABC
A 2 terms
ADR
Architecture Decision Record — a single-page record of one accepted technical decision. Captures Context · Decision · Consequences · Alternatives. See: ADR index
API
Application Programming Interface — the contract a service publishes for other services to call. We render OpenAPI for HTTP, AsyncAPI for events.
B 1 term
BRD
Business Requirements Document — pre-implementation artefact capturing what is being built and why, before technical design.

Markup

<nav class="glossary-azbar" aria-label="Jump to letter">
  <a href="#A">A</a><a href="#B">B</a><a href="#C">C</a>…
</nav>

<section class="glossary-group" id="A">
  <div class="glossary-group__head">
    <span class="glossary-group__letter">A</span>
    <span class="glossary-group__count">4 terms</span>
  </div>
  <dl class="glossary-list">
    <div class="glossary-term">
      <dt class="glossary-term__key">ADR</dt>
      <dd class="glossary-term__def">
        <strong>Architecture Decision Record</strong> — …
        <span class="glossary-term__see">See: <a href="…">ADR index</a></span>
      </dd>
    </div>
    …
  </dl>
      </section>

Anatomy

  • .glossary-azbar — the jump bar; a bordered card with one anchor per letter. Mono font, accent-tinted hover background.
  • .glossary-group — a per-letter section; scroll-margin-top tuned so anchored navigation lands below the sticky topbar.
  • .glossary-group__head — the letter header row.
  • .glossary-group__letter — large mono letter in --accent.
  • .glossary-group__count — small mono caption «N terms».
  • .glossary-list — the wrapping <dl>; grid-spaced rows.
  • .glossary-term — one row. Two-column grid (key · def), bottom hairline separates rows; the last row drops it.
  • .glossary-term__key — the <dt>; mono semibold, fixed 9 rem column on desktop.
  • .glossary-term__def — the <dd>; body text with inline <code> shrunk to 0.9 em.
  • .glossary-term__see — small mono «See: …» link line under the definition. Tone --accent; underline on hover.

Responsive

Below 760 px the term row collapses to a single column — the key sits above the definition with reduced vertical gap. The jump bar wraps freely (no horizontal scroll).

A11y

  • Use a real <dl> with <dt> / <dd>. Screen readers announce «term» / «definition» natively — no role needed.
  • Wrap pairs in <div class="glossary-term"> for grid layout — this is the HTML living-standard pattern for grouping definition rows inside a <dl>.
  • The jump bar should be a <nav aria-label="Jump to letter">; group ids must match the bar's anchor targets.
  • The per-letter <section> needs an id matching the anchor — the bar links to it, and screen readers can navigate with «next section».

Tokens used

  • --accent — letter colour, see-link tone, hover tint.
  • --line, --line-soft — jump-bar border, group head rule, row separators.
  • --bg-elev, --bg-mute — jump-bar surface, hover background.
  • --font-mono — bar, group head, term key, «see».
  • --fs-100, --fs-200, --fs-300, --fs-600 — type scale.
  • --space-1 … --space-6, --radius-xs, --radius-lg — spacing / corners.

Naming note

The legacy class prefix was .gl-* (gl-azbar · gl-group · gl-list · gl-term). The canonical kit prefix is .glossary-*; consumers migrating only need a search-replace. The onboarding glossary at internal/onboarding/glossary.html already uses the canonical names.
