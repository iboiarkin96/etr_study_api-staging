Kind block

Per-kind section block used on catalog reference pages (entity-card, spine-matrix, future per-kind deep-dives). A numbered sub-section title above a tone-tinted strip with a left-edge accent rail, followed by whatever children the page lays out next (a field table, tile grid, anatomy diagram, …).

Example

02

Component

🧩

component

A deployable unit owned by exactly one team — service, worker, or static site. The atomic unit of operational responsibility.

Children go here — a field table, an anatomy diagram, an example markup block.

Kinds (5 tones)

Tone is selected by data-kind="system|component|library|api|resource" on the outer .kind-block; the kit maps each kind to a token and exposes it as --kind-tone — kept distinct from the generic --tone slot so the per-kind palette is stable across pages even if a parent overrides --tone for other reasons.

data-kind="system"

Selector inventory

  • .kind-block[data-kind="…"] — outer wrapper. data-kind selects the tone token and exposes it as --kind-tone.
  • .kind-block__title — numbered sub-section heading above the frame. Combine with .numbered-section__head + .numbered-section__kicker for the mono "01" badge; the kicker is auto-recoloured to the kind tone.
  • .kind-block__head — tone-tinted strip with a left-edge rail (var(--rail-w) solid --kind-tone) carrying the kind's icon + name + role description.
  • .kind-block__icon — emoji or glyph at 1.4rem; visually distinct, semantically decorative.
  • .kind-block__name — the kind's machine name (component, api, …) in --text-strong.
  • .kind-block__role — one-paragraph role description; spans full row via flex: 1 1 100% so it wraps below name on tight widths.

Markup

<section class="kind-block" data-kind="component">
<div class="kind-block__title numbered-section__head">
<span class="numbered-section__kicker">02</span>
<h3>Component</h3>
</div>

<header class="kind-block__head">
<span class="kind-block__icon" aria-hidden="true">🧩</span>
<h4 class="kind-block__name">component</h4>
<p class="kind-block__role">A deployable unit owned by exactly one team.</p>
</header>

<!-- children — field table, tile grid, anatomy diagram, … -->
      </section>

A11y

  • The outer wrapper is a <section> — pair the heading inside __title with aria-labelledby if you want it to read as a landmark.
  • The icon glyph is decorative — keep it in an aria-hidden span so screen readers don't announce the emoji name.
  • Tone is encoded via colour (rail + 4 % wash) and via the data-kind attribute itself; the machine name in __name carries the same information in text — so the meaning never relies on colour alone.
  • The left rail is a border, not a positioned overlay — it doesn't block focus or pointer events and it survives high-contrast / forced-colors mode.

Tokens used

  • --kind-tone (local, set by data-kind) ← --info (system) · --accent (component) · --teal (library) · --success (api) · --warn (resource)
  • --rail-w (canonical 3 px left rail; never hard-coded — see memory note "Accent rail width token")
  • --bg-elev, --line
  • --text-strong, --muted
  • --space-2, --space-3, --space-4, --space-7
  • --radius-md, --fs-300, --fs-500, --fw-semibold
