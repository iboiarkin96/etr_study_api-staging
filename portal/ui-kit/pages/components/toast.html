Toast

Transient capsule notification anchored bottom-right. One toast per action — short, factual, dismissible. Lives ~2.4 s by default, then slides out. Backed by a single live region (aria-live="polite") for screen readers.

Example

Click any button below to show a toast. They stack in the bottom-right corner (bottom of the viewport on mobile).

Variants

  • Neutral — default. Accent halo. Use for informational confirmations (theme switch, copied, saved).
  • Success — green halo + check icon. Use sparingly; most “done” actions are best left silent.
  • Warn — amber halo. Recoverable problem (slow network, rate-limit hint).
  • Danger — red halo. Action failed and needs the user's attention; consider pairing with a retry CTA on the call site (not inside the toast).

API

window.docsToast.show({
message: "Switched to dark theme", // required
variant: "neutral",                 // neutral | success | warn | danger
icon:    "moon",                    // sun | moon | info | check | warn | danger
duration: 2200                      // ms; 0 = persistent until manual dismiss
});

// Returns { dismiss, el } so the caller can close it programmatically.
const t = window.docsToast.show({ message: "Uploading…", duration: 0 });
// later …
        t.dismiss();

Markup (rendered)

<div class="docs-toast-stack" role="region" aria-live="polite" aria-label="Notifications">
<div class="docs-toast docs-toast--neutral" role="status">
<span class="docs-toast__icon"><svg …></svg></span>
<span class="docs-toast__msg">Switched to dark theme</span>
<button class="docs-toast__close" type="button" aria-label="Dismiss">×</button>
</div>
        </div>

The stack element is created lazily on first show() call and reused. You should not embed it manually.

States

  • Entering — 220 ms slide-up + fade-in (--ease-spring), halo opacity 0 → 1.
  • Visible — stays for duration ms. Hover does not pause (kept simple by design; use duration: 0 for sticky toasts that need user dismissal).
  • Leaving — 260 ms fade-out + 6 px upward translate, then DOM removal.

A11y

  • The stack is a single role="region" with aria-live="polite" — screen readers announce each new toast without interrupting the user.
  • Each toast carries role="status" and a × close button with aria-label="Dismiss".
  • prefers-reduced-motion — entrance/exit motion is neutralised globally via tokens/_base.css; opacity transition still applies subtly.
  • Never use toasts for critical errors that require an action — use a modal or inline error instead. Toasts are dismissible by time, so the message is ephemeral.
  • Keep message under ~80 characters. Long messages truncate the experience and crowd the live region.

Tokens used

  • --card, --line, --text, --muted
  • --accent (neutral halo), --success / --warn / --danger (variant halos)
  • --radius-full, --shadow-md
  • --space-1 … --space-5, --fs-300, --lh-snug
  • --ease-out, --ease-spring
