Sidebar

Left-rail navigation for desktop (≥1025 px). One data-driven component for every portal — driven by nav-tree-<portal>.json. Built around three pieces:

  • Brand block — sticky wordmark with a product name, accent fragment, and tagline. Replaces the old "ETR Study API · Project Documentation" header.
  • Full-sidebar collapse — chevron toggle that shrinks the entire sidebar to a 56 px rail. Persisted in localStorage; the active page survives the round-trip.
  • Per-section emoji icons — optional icon field on any node renders an emoji (with aria-hidden) next to the label.

Example · expanded

Live sidebar mounted from nav-tree-uikit.json. The brand block is sticky, every top-level section carries an emoji, and the chevron in the header collapses the whole rail (try it — your choice persists across reloads).

Brand block

Configure the wordmark via the top-level brand object in the nav-tree JSON. All fields are optional; if brand is omitted the header still renders, only the collapse toggle remains.

{
"brand": {
"mark": "🚀",
"productHtml": "Study App · <span class=\"docs-sidebar__brand-product-accent\">UI Kit</span>",
"tagline": "Portal Documentation",
"href": "/services/portal/ui-kit/index.html",
"ariaLabel": "Study App UI Kit — portal documentation"
},
"sections": [ ... ]
      }
  • mark — small emoji or single character rendered in a 32 × 32 accent-tinted square.
  • productHtml — bolded product name; an inner .docs-sidebar__brand-product-accent span paints a fragment in the portal accent.
  • product — plain-text alternative to productHtml.
  • tagline — small-caps eyebrow rendered beneath the product (e.g. "Portal Documentation").
  • href + ariaLabel — wrap the wordmark in a link back to the docs root.

Full-sidebar collapse

The chevron button in the brand row collapses the entire sidebar to a 56 px rail. The toggle:

  • flips aria-expanded on the button and adds .is-sidebar-collapsed to the nearest .docs-shell;
  • rotates the chevron 180° and hides the wordmark and tree;
  • swaps the aria-label between "Collapse navigation" and "Expand navigation" so screen readers and the native browser tooltip announce the correct action;
  • persists state in localStorage["docs-sidebar-shell-collapsed-v2"] so it survives reloads and cross-page navigation.

Per-group caret toggles (the small ▾ on each section row) are independent — they still expand/collapse single subtrees and keep their own state in localStorage["docs-sidebar-expanded-v2"]. Storage holds the set of explicitly-expanded group IDs; the default for a fresh visitor is all groups collapsed. The section containing the active page is always force-expanded so the user can see where they are.

Per-section emojis

Any node may set an icon string (typically an emoji). It renders before the label with aria-hidden="true" so screen readers don't re-announce it. Reserve it for top-level groups — leaf items stay clean.

{
"id": "components",
"label": "Components",
"icon": "🧩",
"children": [ ... ]
      }

Clickable section hubs

A node can act as both a clickable hub and an expandable group at the same time. Set href on the same node that carries children — the row renders as [caret] [icon] [link]. Clicking the label navigates to the hub page; clicking the caret (or the empty area of the row) toggles the subtree. This replaces the older convention of a redundant first child labelled "Hub".

{
"id": "components",
"label": "Components",
"icon": "🧩",
"href": "/services/portal/ui-kit/pages/components/index.html",
"children": [
{ "id": "breadcrumbs", "label": "Breadcrumbs", "href": "…" },
…
]
      }

Behavior:

  • Label click — navigates to the hub (the row uses .docs-sidebar__link; the global row-click handler returns early on links so no toggling happens).
  • Caret click — toggles [data-collapsed] on the <li> and updates localStorage["docs-sidebar-expanded-v2"].
  • Active state — when the current page is the hub URL, the label gets aria-current="page"; when a descendant is active, the section is auto-expanded via hasActiveDescendant() but the label itself stays inactive.
  • When to use — any top-level Diátaxis group (Tutorials, How-to, Reference, Explanation) or operational catalog (Services, Team, Backlog) whose hub page lists or routes to its children. Skip href only when no hub page has been created yet; the label then renders as a non-clickable <span>. The UI Kit's Foundations, Components, and Templates sections all carry hub hrefs today.

States

  • Item · default — --muted text on --bg.
  • Item · hover — background --bg-mute, text --text.
  • Item · active — aria-current="page" on the link; accent-tinted row, accent left bar (3 px), bold weight.
  • Group · collapsed — [data-collapsed="true"] on the <li>; caret rotates -90°, <ul.docs-sidebar__children> is hidden. This is the default for any group that hasn't been explicitly expanded by the user (or whose subtree doesn't contain the active page).
  • Group · expanded — caret pointing down; either the user clicked the caret/row, or the section contains the active page.
  • Group · hub link — when the section node carries both href and children, the row is [caret] [icon] [link]: the label navigates, the caret toggles. See Clickable section hubs.
  • Shell · collapsed — .docs-shell.is-sidebar-collapsed shrinks the sidebar to 56 px, hides the wordmark + tree, and rotates the toggle chevron 180°. The button's aria-label flips to "Expand navigation".

A11y

  • Use <aside aria-label="…"> as the wrapper so screen readers announce a complementary landmark.
  • The brand block is a <header>; its toggle is a <button> with aria-expanded and an accessible label that flips between "Collapse navigation" and "Expand navigation".
  • Group toggles are also <button>s with aria-label="Toggle section"; their expanded state lives on the parent <li data-collapsed>.
  • Emoji icons are decorative and must carry aria-hidden="true" — the label text remains the only announcement.
  • Current page must be marked with aria-current="page" in addition to the visual active styling.
  • Focus ring on items must remain visible — never set outline:none without a replacement.
  • Mobile drawer (see drawer) replaces this component below 1025 px and must visually mirror it (per memory).

Tokens used

  • --layout-sidebar (280 px, expanded width)
  • --layout-sidebar-collapsed (56 px, collapsed rail width)
  • --bg, --bg-mute, --line, --line-soft
  • --accent (active bar, brand mark tint, accent fragment of the wordmark)
  • --text, --muted, --muted-soft
  • --fs-100, --fs-300, --fw-semibold, --fw-bold
  • --radius-sm, --radius-md, --radius-full
  • --ease-out (collapse + group transitions)

Persistence keys

  • localStorage["docs-sidebar-shell-collapsed-v2"] — "1" or "0"; full-sidebar collapsed state.
  • localStorage["docs-sidebar-expanded-v2"] — JSON array of explicitly-expanded group ids. Default (no entry) means every group is collapsed; the section containing the active page is still force-expanded at render time.
  • sessionStorage["docs-sidebar-scroll-v2"] — last sidebar scroll position (skipped inside drawer copies).
