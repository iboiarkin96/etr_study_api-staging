Filter chips

JS-only behaviour. Wires a chip group to a filterable container and a URL hash. Single-axis filtering — one active chip at a time, one tag matched at a time. The chips themselves are .docs-chip--filter from chip.css; this page documents the controller (filter-chips.js).

Example

  • Rewrite onboarding guide docs
  • Add Backstage entity links infra
  • Polish search empty state ux
  • Document runbook lifecycle docs infra

Items with [data-filter-hidden] are visually hidden by chip.css.

Mount

<div data-filter-group data-target="#tickets" class="docs-chip-row" aria-label="Filter tickets">
  <button class="docs-chip docs-chip--filter" data-filter="all">All</button>
  <button class="docs-chip docs-chip--filter" data-filter="docs">Docs</button>
  <button class="docs-chip docs-chip--filter" data-filter="infra">Infra</button>
</div>

<ul id="tickets">
  <li data-tags="docs">…</li>
  <li data-tags="docs infra">…</li>
  <li data-tags="infra">…</li>
      </ul>

Attributes

  • [data-filter-group] — marker on the chip container. Boolean.
  • [data-target] — CSS selector of the filterable container (queried with document.querySelector). Required.
  • [data-filter="…"] on each chip — the tag this chip activates. Use "all" to show everything.
  • [data-tags="tag1 tag2"] on each item — space-separated tag list. An item is shown when its tag list contains the active filter (or the filter is "all").
  • aria-pressed on the chip — set automatically by the controller; the active chip also gets .docs-chip--active.

State & URL hash

  • Active filter is reflected in the URL as #filter=<tag>.
  • "all" is treated as absence — the hash is cleared so shareable URLs without filters stay tidy.
  • On load, the controller reads the hash; if it matches a known chip, that filter becomes active. Otherwise, the first chip (typically data-filter="all") is selected.

Single-key vs multi-key

This component owns one axis of filtering at a time. When a page needs multiple cooperating axes sharing a hash (e.g. #sprint=…&profession=…&status=…), reach for multi-filter-chips instead. multi-filter-chips also dispatches a window-level multifilterchange event that other modules (like the backlog-cockpit renderer) can listen to.

A11y

  • Chips are real <button> elements with aria-pressed reflecting the active state.
  • The group should carry an aria-label describing what is being filtered (e.g. aria-label="Filter tickets").
  • Hidden items get the [data-filter-hidden] attribute — content remains in the DOM so screen-reader users can still navigate forward/backward into the full list without the chip handler running.
