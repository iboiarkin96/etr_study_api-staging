Rocket

Fixed scroll-to-top FAB that appears once the reader passes 300 px of scroll. A conic-gradient progress ring wraps the button border and fills 0 → 100 % as the page scrolls. Five ember particles rise continuously from the exhaust; three CSS smoke puffs drift downward. On click, 800 ms of WebGL2 FBM smoke pre-charges before the rocket flies a G2-continuous Bézier loop trajectory off-screen — falling back to Canvas2D when WebGL2 is unavailable. Respects prefers-reduced-motion.

Example

Interactive demo — drag the slider to simulate scroll progress, click Launch (or the rocket itself) to watch the full CSS animation. The WebGL smoke fires only on real doc pages where the full entry.js runs.

Anatomy

The FAB stacks several decorative layers under a single accessible <button>. The shell is a 52 × 52 px rounded square; everything inside is aria-hidden chrome that the runtime drives via CSS variables, classes, and per-particle keyframes.

  1. Shell (.docs-rocket-fab) — accessible button with aria-label="Back to top". data-visible="true" after the reader scrolls past 300 px; before then the button is opacity-0 and pointer-events:none.
  2. Progress ring (.docs-rocket-fab__progress-ring) — conic-gradient stroke wrapping the border, driven by the CSS custom property --rocket-progress: 0 → 100. Updated on scroll as scrollY / (scrollHeight − innerHeight) × 100. Purely decorative (pointer-events:none).
  3. Rocket glyph (.docs-rocket-fab__rocket) — stroke-only inline SVG (viewBox 0 0 24 24, stroke 1.8 px, currentColor). Four paths: hull, porthole, fins, nozzle bar. Idle 1.7 s bob + rotate; accelerates to 1 s on hover.
  4. Ember particles (.docs-rocket-fab__ember--1 … --5) — five small accent dots rising from the exhaust on staggered loops. Pure CSS, no JS per-frame work.
  5. Smoke puffs (.docs-rocket-fab__smoke--left|mid|right) — three CSS blurs drifting downward; background is --card mixed with white for a soft cloud read.
  6. Flame (.docs-rocket-fab__flame) — appears only with .is-launching; pulses while the WebGL2 smoke pre-charges. Hidden during idle / hover.
  7. Trail + speed lines (.docs-rocket-fab__trail, .docs-rocket-fab__speed-line--a, --b) — radial streaks that fade in during the launch phase to sell the Bézier exit.
  8. Rocket wrap (.docs-rocket-fab__rocket-wrap) — the span the JS targets with transform for the off-screen flight. The shell stays in place; only the wrap moves.

Animation lifecycle

  1. Mount. mountRocket() detects body.docs-shell and injects the FAB before the closing </body>. On landing pages it instead loads the home WebGL flowfield into any [data-component="rocket"] slot — the same export serves both surfaces.
  2. Reveal. A scroll listener flips data-visible when window.scrollY ≥ 300. The same handler updates --rocket-progress on every frame.
  3. Idle. Bob (Y-axis translate) + slight Z-rotate at 1.7 s. Five ember loops + three smoke loops run continuously.
  4. Hover / focus. Border tints to accent, translateY(-2 px), --shadow-md lifts elevation; bob accelerates to 1 s. :focus-visible draws a 2 px accent ring at 2 px offset.
  5. Click → pre-charge (800 ms). Class .is-launching fades the shell to transparent; the CSS flame pulses. In parallel a WebGL2 canvas is created with an FBM (domain-warped fractal Brownian motion) smoke shader; particles begin spawning before the rocket moves.
  6. Flight (2 600 ms). The rocket-wrap follows a G2-continuous Bézier loop: approach segment → full circle → exit. A C2-smooth velocity profile (smootherstep + Gaussian apex) drives a per-frame tangent rotation, curvature banking, and speed stretch. Particle pool is 2400 on WebGL2 / 800 on the Canvas2D fallback.
  7. Settle. window.scrollTo({ top: 0, behavior: "smooth" }) fires near the apex so the page snaps back while the rocket is still off-screen. The class is removed; idle bob resumes.
  8. Reduced motion. When prefers-reduced-motion: reduce is set, all CSS keyframes are disabled and the click handler short-circuits to window.scrollTo({ top: 0 }) with no WebGL canvas created.

A11y

  • Uses a real <button type="button"> — activatable with Space / Enter.
  • aria-label="Back to top"; all inner children are aria-hidden="true".
  • :focus-visible shows a 2 px --accent ring at 2 px offset; suppressed for mouse clicks.
  • When data-visible="false" the button has opacity:0; pointer-events:none — it remains in the accessibility tree, so add tabindex="-1" if needed.
  • All animations — ember rise, bob, smoke puffs, flame — are disabled under prefers-reduced-motion: reduce. The JS click handler checks the media query before starting WebGL.
  • Progress ring is decorative only (pointer-events:none); do not rely on it as the sole position indicator.

Tokens used

  • --accent — button color, progress ring arc, focus ring, hover border tint
  • --card — button background; CSS smoke background (mixed with white)
  • --line — button border; progress ring track
  • --bg-mute — hover background
  • --radius-full — button and progress-ring border-radius
  • --shadow-sm / --shadow-md — resting / hover elevation
  • --space-3 / --space-5 — position offsets mobile / desktop
