Roles radar

Page-template component. Renders the «25-practices · AS-IS vs TO-BE» radar page used by every SDLC role hub (SA · Architect · Dev · Manager · QA · SRE). One full-bleed SVG radar on top, a 5-column index of practice names below, then five group sections (A–E) with per-group mini-radars and practice cards. Different shape from the generic .docs-radar component — see «When to pick which» below.

Anatomy — what the page contains

  1. Hero (.hero) — eyebrow chip + role title + lede.
  2. Main radar block (.main-radar-block) — wide SVG with 25 axes, two polygons (AS-IS rose / TO-BE teal), numbered group-colored badges (gA…gE) at each axis tip, and an interactive sticky aside (.main-radar-aside) with legend, summary, and «read this chart» tip.
  3. Number legend (.number-legend) — 5-column grid mapping the 25 numbers to practice names.
  4. Group sections (.group-section, five total) — each has a sticky mini-radar showing just that group's axes, and a scrolling column of practice cards.
  5. Practice cards (.practice-card) — title link, AS-IS/TO-BE scores, one-line description, group accent stripe.

CSS namespace

All selectors are scoped under .roles-radar-page on the <body> so the page-template can't leak into siblings. 133 selectors, 690 lines — file lives at services/frontend/portal/assets_v2/ui-kit/components/roles-radar.css and ships in entry.css.

Group tones come from CSS variables (--gA…--gE) declared on .roles-radar-page. Override the palette by setting them on the body or a wrapper — the SVG badges, legend swatches, and card stripes all read from there.

When to pick which

  • Use .roles-radar-page only for the 6 role-radar pages — it's a full page template with hand-authored SVG carrying 25 axes + per-group interaction.
  • Use the generic .docs-radar component for everything else — JS-mounted from data-axes, auto-renders SVG + legend + scroll-spy, works at any axis count.

Adding a new role

  1. Copy one of the six live exemplars to internal/team/roles/<new-role>/radar.html.
  2. Keep the class="roles-radar-page" on the body and the SVG markup intact.
  3. Edit the 25 practice axes + AS-IS/TO-BE numbers; update the legend names.
  4. Register the page in nav-tree-internal.json under team/roles/<role>/.

No CSS edits needed — the template inherits everything from roles-radar.css.
