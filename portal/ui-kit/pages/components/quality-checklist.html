Quality checklist

Vertical list of rubric items, each with a square mark on the left and a title + hint on the right. Three states: done (green ✓), partial (amber ~), todo (muted ○). Use anywhere the reader needs to walk a rubric — practice maturity, ratification gates, audit findings, definition-of-done checks.

Example

  • ✓

    Test pyramid documented in the repo

    A page exists explaining unit / integration / system ratios; reviewers cite it.

  • ~

    Contract-test coverage for every public endpoint

    ~70% of endpoints have Schemathesis baseline; gaps are tracked.

  • ○

    Mutation testing in CI

    Pilot on one service first; expand once the score ratchet is wired up.

Anatomy

  • .pa-quality — wrapper <ul>, no list bullets, no padding.
  • .pa-quality__item — bordered card; pair with exactly one state modifier:
    • .pa-quality__item--done — green tinted mark
    • .pa-quality__item--partial — amber tinted mark
    • .pa-quality__item--todo — muted mark
  • .pa-quality__mark — 28×28 square mark cell (✓ / ~ / ○ glyph). Always provide an aria-label.
  • .pa-quality__title — short imperative sentence (the bar to meet).
  • .pa-quality__hint — one-liner explaining the bar or current status.

Markup

<ul class="pa-quality">
  <li class="pa-quality__item pa-quality__item--done">
    <span class="pa-quality__mark" aria-label="done">✓</span>
    <div>

      <p class="pa-quality__hint">…</p>
    </div>
  </li>
      </ul>
