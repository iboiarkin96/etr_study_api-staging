Drawer

Mobile navigation drawer (≤1024 px). Triggered from the topbar burger, slides in from the left with a backdrop. Per the project memory, the drawer must visually mirror the desktop sidebar — same hierarchy, same widths, not a body-sized "native list view".

Example

Static demo of the open state. The real .docs-drawer class is visibility:hidden + display:none on desktop, so this preview uses inline styles to mirror the visual.

Variants

  • Default — slide-in from left, backdrop fade-in.
  • Right-side — same component with .docs-drawer--right modifier; used for TOC FAB on mobile.
  • Full-height — covers from top: 0 below topbar.

States

  • Closed — translated off-screen, hidden attribute set.
  • Opening — slide-in 200 ms with --ease-out.
  • Open — full opacity; focus moved to close button; body scroll locked.
  • Closing — slide-out 160 ms with --ease-in-out; focus returned to trigger.

A11y

  • Use role="dialog" and aria-modal="true"; trap focus inside while open.
  • ESC closes the drawer; return focus to the trigger button.
  • Backdrop click closes; backdrop has aria-hidden="true".
  • Lock body scroll while open (overflow:hidden on <html>).
  • Trigger has aria-expanded and aria-controls pointing at drawer id.

Tokens used

  • --layout-sidebar, --bg-elev, --line, --shadow-lg
  • --ease-out, --ease-in-out
