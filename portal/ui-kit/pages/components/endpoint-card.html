Endpoint card

Collapsible API endpoint card — method badge + path + optional summary, expanding to reveal request and response code blocks. Used in doc-reference templates to replace flat <h3> + code blocks.

Example

Three endpoints — click a header to expand

GET /api/v1/sessions/{id} Fetch a single session by ID

Returns the full session object or 404 if not found or not owned by the requesting user.

Response 200

json{
"id": "0e5b…",
"started_at": "2026-05-11T18:24:00Z",
"cards_reviewed": 24
                }
POST /api/v1/sessions Open a new study session

Creates a new session. Server fills id and started_at.

Response 201

json{ "id": "1a7c…", "started_at": "2026-05-14T09:00:00Z" }
DELETE /api/v1/sessions/{id} Delete a session record

Permanently deletes the session. Returns 204 No Content on success.

JS (mountEndpointCards() in entry.js) attaches click + keyboard handlers automatically to every .docs-endpoint-card found at boot. Set data-open="true" to pre-expand.

States

StateAttributeVisual
Collapsed (default)data-open="false"Header only; chevron points down; no shadow
Expandeddata-open="true"Body visible with grid-template-rows animation; shadow-sm; chevron rotates 180°
Hover (header)—--bg-mute background on header
Focus (keyboard)—2px --accent outline inset on header

Accessibility

JS sets role="button", tabindex="0", and aria-expanded on the header. The body has role="region". Keyboard users can toggle with Enter or Space. The chevron SVG is aria-hidden="true".

Tokens used

--card, --line, --bg-mute, --text, --muted, --accent, --shadow-sm, --radius-md, --font-mono, --fs-100, --fs-200, --fw-medium, --fw-semibold, --space-2 through --space-4
