Signal pillars

Coordinated row of parallel signal cards with a tone-tinted glyph banner, a one-line lede, and a three-row fact list. Optional shared-key ribbon sits under the row to mark items that share a common identifier. Pairs with the separate .docs-probes block for items that look similar but belong to a different category.

Different category

Health endpoints — active probes

Not telemetry. Boolean checks the orchestrator calls to decide whether the process should keep receiving traffic.

  • /healthz “The process is alive.”
  • /readyz “The process can serve traffic” — DB up, migrations done, dependencies reachable.

Example — four-card severity variant

Add --four to the row to switch from a 3-column to a 4-column grid. Use this for severity ladders (SEV-1..4) and any other parallel classification where four items make sense. At tablet widths (≤1024 px) the grid folds to 2×2; at mobile (≤760 px) it collapses to a single column.

  1. SEV-1
    Major outage

    Service unusable for most users, or data at risk.

    Postmortem
    Required
  2. SEV-2
    Partial outage

    Core feature down or seriously slow for many users.

    Postmortem
    Required
  3. SEV-3
    Degraded

    Noticeable but not blocking.

    Postmortem
    Recommended
  4. SEV-4
    Minor

    Annoying — can wait until the next working day.

    Postmortem
    Not required

Tones & glyphs

data-tone swaps the per-card accent (glyph banner tint, glyph stroke/fill colour via currentColor, link colour, top border). Use the tone that matches the signal's pill.

  • data-tone="accent" — --accent.
  • data-tone="teal" — --teal.
  • data-tone="rose" — --rose.
  • data-tone="info" — --info.
  • data-tone="danger" — --danger (SEV-1).
  • data-tone="warn" — --warn (SEV-2).
  • data-tone="success" — --success.
  • data-tone="neutral" — --muted-soft (SEV-4, “different category”).

The __link slot accepts either a real <a> (for the 3-card pillars where each card points at a sibling page) or a non-interactive <span> (used as a subtitle on the severity variant — “Major outage”, “Partial outage”, etc.). Same CSS, no rename.

The glyph is an inline <svg> inside .docs-signal-pillars__glyph. The wrapper sets color: var(--pillar-tone); the SVG uses fill="currentColor" / stroke="currentColor" so it inherits the tone automatically. Default canvas is viewBox="0 0 80 32"; the wrapper renders it at max-width: 140px, max-height: 40px.

Shared-key ribbon

The ribbon (__ribbon) is optional and decorative — its job is to visually say “these three share something”. Two small tick marks (::before / ::after) connect the ribbon up to the cards above. The __ribbon-key span renders a monospace code chip. Hide the ribbon if the cards do not share a common identifier — there is no semantic loss.

Probes block

.docs-probes is a sibling component that pairs with signal-pillars. Use it for items that look superficially similar (one-line endpoints with a short answer) but belong to a different category — e.g. active probes alongside passive telemetry. The 3 px muted left border and grey surface signal “separate group”; the “Different category” chip makes the boundary explicit.

Responsive behaviour

  • ≥1025 px — default row is three columns; --four row is four columns. Full padding, ribbon tick-marks visible.
  • 761–1024 px — default row stays three columns; --four row folds into 2×2. Padding tightened, glyph banner 56 px.
  • ≤760 px — both rows collapse to a single column. Ribbon ticks disappear, ribbon spans full width. Probes list also collapses to one column.

A11y

  • The row is an <ol> with an aria-label describing the family (e.g. “Three telemetry pillars”).
  • Glyph banners are aria-hidden="true"; the pill carries the signal name and is the accessible label.
  • The shared-key ribbon is aria-hidden="true" — its purpose is purely visual foreshadowing.
  • Component-scoped selectors raise specificity so prose.css defaults (p / ol / dl bottom margins, ol left padding) lose the cascade fight.

Tokens used

  • --accent, --teal, --rose, --info, --danger, --warn, --success, --muted-soft — per-pillar tones.
  • --card, --bg-elev, --bg-mute, --line — card surface, glyph banner, probes surface.
  • --text-strong, --text, --muted, --muted-soft — lede, facts, ribbon copy, fact keys.
  • --shadow-sm, --radius-lg, --radius-md, --radius-xs.
  • --font-mono, --fs-100 … --fs-500, --fw-semibold.
  • --space-1 … --space-6.
