Author chip

Contributor pill that resolves a data-person-id against the maintainers directory and renders an avatar + name (+ optional handle / role). Used in every page's <footer class="docs-history"> row and in author bylines.

Example

Three live chips resolved against maintainers.json via the JS mount. Hover for the native title (name — role).

@ivan @ivan @ivan

Variants

  • data-variant="sm" — 20 px avatar, name only. Used in compact history rows and inline bylines.
  • default (omit attribute) — 32 px avatar, name + handle + role. Used in author cards and dense bylines.
  • data-variant="xl" — 48 px avatar with soft shadow, larger name (--fs-500) + meta. Used in page-head bylines and people profile pages.

Attributes

  • data-person-id — 32-char hex key in the maintainers directory (preferred).
  • data-person-slug — slug fallback (e.g. ivan-boyarkin).
  • data-handle — @iboiarkin96-style handle fallback.
  • data-variant — "sm" | "" | "xl" (default empty).
  • data-show-meta="false" — hides handle / role on the default + xl variants.
  • data-src — override the maintainers JSON URL (rare; used by isolated previews).

Markup

Render an empty anchor — the JS mount fills the avatar, name, and meta from the lookup. If the lookup misses, the chip renders a placeholder with data-state="missing".

<a class="docs-history__author"
   data-component="author-chip"
   data-person-id="16fc8b78537109162984a2fdbef6e142"
      data-variant="sm">@ivan</a>

Inside a history row:

<li class="docs-history__row">
<span class="docs-history__date">2026-05-25</span>
<a class="docs-history__author"
data-component="author-chip"
data-person-id="16fc8b78537109162984a2fdbef6e142"
data-variant="sm">@ivan</a>
<span class="docs-history__note">Initial showcase.</span>
      </li>

States

  • Loading — data-state="loading", opacity 0.5 while the maintainers JSON resolves.
  • Ready — data-state="ready", anchor href points at the person profile.
  • Missing — data-state="missing", renders "Unknown contributor" + ? avatar fallback.
  • Hover / focus — pill background switches to --bg-mute, gains 1 px inset ring on --line.
  • Active — translateY(1 px) micro-press.

A11y

  • The chip is a real <a> — keyboard-reachable, focusable, navigable to the person's profile page.
  • Focus ring is a 2 px --accent outline at 2 px offset (visible on both surfaces).
  • Avatar <img> has empty alt="" — the name string next to it is the accessible label, so screen readers don't double-announce.
  • On image load failure the avatar swaps for a text-initials fallback (still aria-hidden).
  • The hover title attribute carries the full "Name — Role" — readable by AT but not relied on as the sole label.
  • prefers-reduced-motion: reduce drops the hover transition.

Tokens used

  • --bg-mute, --line (hover surface + ring)
  • --accent (focus outline, fallback-initial colour)
  • --text-strong, --muted (name vs. meta)
  • --radius-full, --space-1..--space-4
  • --fs-200, --fs-300, --fs-500
  • --ease-out (transitions)
