Section card

Neutral bordered container for a chunk of long-form content — what/why/how explanation, a single rubric, a methodology block. Lighter sibling of .docs-card: no elevation, no hover, no tone rail — just a frame plus tightened inner-prose rhythm. Use when content needs visible separation without competing with adjacent kit cards.

Example

Why this practice exists

Without a written contract for «when do we cut a release branch?» the answer lives in three engineers' heads, gets misremembered under pressure, and silently rots as the team rotates.

The contract in one paragraph

Branch is cut the Tuesday before the release week; only fixes for that release land on it; everything else continues on main. The branch dies after the release ships.

What this isn't

  • Not a long-lived develop branch. We use trunk-based development.
  • Not a hotfix mechanism — those branch off the release tag, not off the cut.
  • Not a place for half-finished features to hide.

Anatomy

  • .pa-section — bordered wrapper, --bg-elev background, large padding.
  • Direct > h2 — semibold section title, 1.25rem, no top margin.
  • Direct > h3 — semibold sub-heading, 1rem, top margin only.
  • p — line-height 1.6, bottom margin reset on last child.
  • ul / ol — left padding 1.3em, items breathe by 4 px.
  • .docs-table-scroll — companion wrapper for wide tables inside the section; keeps horizontal overflow scrollable without breaking the section frame.

Markup

<section class="pa-section">
  <h2>…</h2>
  <p>…</p>
  <h3>…</h3>
  <p>…</p>
  <ul>…</ul>

  <!-- optional wide table -->
  <div class="docs-table-scroll" role="region" aria-label="…">
    <table class="docs-table">…</table>
  </div>
      </section>

vs .docs-card & .docs-card--rail

  • .docs-card — interactive tile (hover lift, optional link wrap, footer arrow). Use as a navigation target or a stat tile.
  • .docs-card--rail[data-tone] — same tile + a tone-coloured left rail and (optionally) a tinted background. Use to encode category.
  • .pa-section — flat content panel; no hover, no rail. Use to group prose that belongs together inside an article.

Naming note

Class name uses the legacy .pa-section prefix from practice-article.css; neutral rename to .section-card queued.
