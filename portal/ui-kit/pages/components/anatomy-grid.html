Anatomy grid

Auto-fit grid of small «part» cards — each with a part name and one-line hint. Use to label the moving pieces of a diagram, the slots of a template, the fields of an entity card, the regions of a page skeleton. Pairs naturally with an inline-SVG figure above where each part is numbered.

Example

① Title

The entity's metadata.name — kebab-case, unique within the catalog.

② Lede

One sentence: «<Name> is <role> — the only artefact that …»

③ Pills row

Lifecycle pill first (colour-coded), then up to 3 tech/spec pills.

④ Fields list

Required fields first, optional after. Owner is always row 1; Lifecycle is row 2.

⑤ Dep-lanes

Structured «Depends on» — grouped chips per category (Framework / Persistence / …).

⑥ External links

Repo · Code reference · Database tables · Public docs — last block in the card.

Anatomy

  • .pa-anatomy — auto-fit grid, minimum 180 px per tile, gap from --space-3.
  • .pa-anatomy__part — bordered card, --bg-elev background.
  • .pa-anatomy__name — short label (often prefixed with a numbered marker ①, ②, … to match a diagram).
  • .pa-anatomy__hint — one-line description, muted.

Markup

<div class="pa-anatomy">
  <div class="pa-anatomy__part">
    <p class="pa-anatomy__name">① Title</p>
    <p class="pa-anatomy__hint">…</p>
  </div>
  <!-- repeat for each part -->
      </div>

Naming note

Class names use the legacy .pa-anatomy prefix; neutral rename to .anatomy-grid is queued.
