Numbered step list

Ordered list rendered as a stack of accent-railed cards with a mono «01 · 02 · 03 …» counter in the left column. Use for required sequences where each step must be visibly numbered and fit on a single card row — page-skeleton contracts, publishing flows, escalation sequences.

Example

  1. Topbar & breadcrumb. The shared <header class="topbar"> with the breadcrumb trail ending at the current page.
  2. Sidebar mount. <aside data-component="sidebar"> — sidebar JS hydrates it.
  3. Hero / page-head. <header class="page-hero page-head"> with eyebrow, <h1>, optional CTAs.
  4. Body sections. One topic per <section class="numbered-section">.
  5. Page-history footer. <footer class="docs-history"> with one row per meaningful edit.

Markup

<ol class="numbered-step-list">
  <li>
    <div><strong>Step title.</strong> Body…</div>
  </li>
  <li>
    <div><strong>Step title.</strong> Body…</div>
  </li>
      </ol>

The counter is rendered by CSS (counter-reset: step + counter-increment: step). Numbers stay in sync with DOM order, so adding or removing an item re-numbers the rest automatically. Decimal-leading-zero padding («01») keeps the column width stable from 01 through 99.

Anatomy

  • .numbered-step-list — wrapping <ol>; resets list bullets and seeds the counter.
  • .numbered-step-list > li — the card row. 2-column grid (counter · body), accent rail on the left, neutral border on the other three sides, --bg-elev surface.
  • ::before — the counter glyph. Mono, bold, accent-tinted.
  • Item body — typically a <div> with one <strong> lead followed by prose. Inline <code> gets a small pill background out of the box.

When to use

  • Numbered step list — required ordered sequence, each step short enough to fit on one card row. Examples: page skeleton, publishing flow, on-call escalation.
  • Rollout timeline — phases over time, each with a status and a date. Use when «when» matters.
  • State strip — finite-state machine; transitions matter more than ordering.
  • Anatomy grid — parts of a thing; order isn't load-bearing.

Tokens used

  • --accent — counter colour + left rail.
  • --rail-w — left-rail width (3 px).
  • --line, --bg-elev — card border + surface.
  • --font-mono, --fs-200, --fw-bold — counter typography.
  • --bg-mute, --radius-xs — inline <code> pill.
  • --space-1 … --space-4, --radius-md — spacing / corners.
