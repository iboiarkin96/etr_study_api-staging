Example frame

A single-purpose wrapper (.docs-example) used on every showcase page to visually separate the live demo from the prose that surrounds it. 1 px border, --radius-lg corners, --space-5 padding, and --space-6 bottom margin to guarantee at least 32 px of air before the next <h2>. The frame is meta: this page demos itself.

Example

This block is itself a .docs-example. The 1 px border and rounded corners separate it from the prose above and below. Live demos for buttons, chips, footers, modals, etc. all live in this wrapper.

Markup

<!-- Canonical pattern on every showcase page -->
<section aria-labelledby="x-ex">
<h2 id="x-ex">Example</h2>
<div class="docs-example">
<!-- optional -->
<p class="docs-example__caption">Short context line.</p>

<!-- demo content -->
<button class="docs-btn">Click me</button>

<!-- optional -->
<p class="docs-example__foot">End of example</p>
</div>
</section>

<!-- Variants -->
<div class="docs-example docs-example--hush">…</div>
      <div class="docs-example docs-example--flush">…</div>

States

  • Default — 1 px --line border, --card background, --radius-lg corners, --space-5 padding, --space-6 bottom margin. First/last child margins are reset so the frame's padding governs spacing.
  • --hush — transparent background, dashed border. Use for demos that need the page bg.
  • --flush — padding stripped, overflow: hidden so the inner demo's corners clip cleanly to the frame's radius.
  • Mobile (≤760 px) — padding softens to --space-4 so the demo doesn't crowd a narrow viewport.

A11y

  • The frame is purely visual — it adds no roles, no labels. The surrounding <section aria-labelledby="x-ex"> still carries the semantics.
  • Eyebrow uses <span>, never an <h*> — keeps the page outline clean. If you need an actual heading, leave the outer <h2> in place above the frame.
  • The caption is a plain <p> — screen readers read it in document order before the demo body.
  • Do not wrap an entire section's multiple example blocks in a single frame; one frame = one cohesive demo so the boundary stays meaningful.

Tokens used

  • --card, --line, --line-soft (foot divider), --muted, --muted-soft
  • --radius-lg
  • --space-3 / --space-4 / --space-5 / --space-6
  • --fs-100 (eyebrow + foot), --fs-300 (caption)
  • --lh-snug
