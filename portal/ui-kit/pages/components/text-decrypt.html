Text decrypt

Matrix-style glyph-scramble decrypt animation for landing-hero <h1>s. Each character cycles through junk glyphs (█▓▒░A-Z#@%&*+=<>/_), then locks into its final letter with an accent-tinted flash. Fires once on first intersection.

Example

This headline decrypts on scroll-into-view — reload the page to replay.

Build, ship, and
document fearlessly.

The second line uses data-decrypt-delay="240" so it starts a quarter-second after the first.

Attributes

  • data-decrypt — required. The final string to decrypt to. Falls back to el.textContent.trim() if empty.
  • data-decrypt-delay — ms to wait after the element enters view before starting (default 0).
  • data-decrypt-duration — per-character scramble window in ms (default 580).
  • data-decrypt-stagger — ms between adjacent characters starting their scramble (default 32).

The kit also exports mountVariableWeight() for [data-variable-weight] — a sibling scroll-driven animation on the same hero <h1> that interpolates font-variation-settings: "wght" from 500 → 820 as the element scrolls past the viewport top.

Markup

<h1 data-variable-weight>
<span data-decrypt="Build, ship, and">Build, ship, and</span>
<span data-decrypt="document fearlessly."
        data-decrypt-delay="240">document fearlessly.</span>
      </h1>

Each data-decrypt span pins its own bounding rect before clearing its text — so wider scramble glyphs cannot reflow the line and shove blocks below it down each frame. white-space: nowrap; overflow: hidden; from the CSS reinforces this.

States

  • Pre-mount — element renders its authored final text (server-rendered, SEO-safe).
  • Scrambling — each char wrapped in <span class="docs-decrypt-char docs-decrypt-char--scrambling">, painted in --accent with a soft text-shadow halo.
  • Settled — span loses --scrambling, returns to inherited colour. data-done="1" on the span.
  • Fires once — IntersectionObserver disconnects after the first 0.3-threshold crossing, then a one-shot setTimeout(delay) kicks the RAF loop.

A11y

  • Before scrambling starts the JS sets aria-label to the final string, so screen readers always announce the real headline — never the junk glyphs.
  • prefers-reduced-motion: reduce short-circuits the animation entirely: the JS sets textContent = finalText and returns. The companion mountVariableWeight() also returns early.
  • The CSS also nulls the accent flash + text-shadow on the scrambling class under reduced motion, in case the JS hasn't bailed yet.
  • The headline remains a real <h1> in the document outline — heading-navigation works whether the animation has run or not.
  • Browsers without IntersectionObserver start immediately (graceful degrade).

Tokens used

  • --accent (scrambling char colour + text-shadow halo via color-mix)
  • --ease-spring (variable-weight transition)
  • CSS variable --docs-h1-wght on each [data-variable-weight] node (500–820, JS-driven)
  • Constants — duration: 580 ms, stagger: 32 ms, IO threshold 0.3
