TL;DR callout

Labelled summary box for the top of a long article — a tone-tinted gradient panel with a mono uppercase «TL;DR» chip and 2–4 sentences of prose. Use to give the reader the answer before they decide to scroll.

Example

TL;DR

An ADR (Michael Nygard, 2011) is a short numbered file with three sections: Context, Decision, Consequences. Permanent integer id (0001, 0042…), status (proposed / accepted / superseded), stored in the repo next to code. Never deleted — only superseded by a later ADR.

Anatomy

  • .pa-tldr — grid wrapper: single column on mobile, 200px · 1fr two-column above 720 px. Tone-tinted gradient background and matching border.
  • .pa-tldr__label — mono uppercase chip in the tone color (typically «TL;DR» or «Summary»).
  • .pa-tldr__text — body prose; uses --fs-500 so it reads slightly larger than the surrounding paragraphs.

Markup

<aside class="pa-tldr">
  <span class="pa-tldr__label">TL;DR</span>
  <p class="pa-tldr__text">…</p>
      </aside>

Tone override

The gradient + label colour read from --pa-accent, which falls back to --accent. On a practice article the wrapper sets --pa-accent via the group class (.pa--gA…gE). To re-tint manually:

<aside class="pa-tldr" style="--pa-accent: var(--success)">
  <span class="pa-tldr__label">Verified</span>
  <p class="pa-tldr__text">…</p>
      </aside>

Naming note

Class names use the legacy .pa-tldr prefix from practice-article.css where this pattern first lived. A neutral rename to .tldr-callout is queued — happens when we mass-migrate the 150 practice pages in one sweep. Until then the class is the contract.
