View switcher

Stripe-style segmented control for switching between named view modes. Persists selection in localStorage, syncs data-view on the target container, and supports arrow-key navigation.

Example

Board / Table / Timeline

Markup

html<div data-component="view-switcher"
data-target="#my-container"
data-storage-key="docs-cockpit-view"
data-default="board"
aria-label="View">
<button class="docs-view-switcher__btn" data-view="board">Board</button>
<button class="docs-view-switcher__btn" data-view="table">Table</button>
<button class="docs-view-switcher__btn" data-view="timeline">Timeline</button>
</div>

<div id="my-container" data-view="board">
<div class="my-view my-view--board">...</div>
<div class="my-view my-view--table">...</div>
<div class="my-view my-view--timeline">...</div>
</div>

<style>
.my-view { display: none; }
[data-view="board"]    .my-view--board    { display: grid; }
[data-view="table"]    .my-view--table    { display: block; }
[data-view="timeline"] .my-view--timeline { display: block; }
      </style>

Data attributes

AttributeRequiredDescription
data-targetYesCSS selector for the container that receives data-view="…"
data-defaultNoInitial view name if no localStorage entry (defaults to first button)
data-storage-keyNolocalStorage key (default: docs-view)
aria-labelRecommendedAccessible name for the role="group" container

Accessibility

JS sets role="group" on the wrapper and aria-pressed on each button. Arrow keys (← →) cycle between options. The active button gets aria-pressed="true". The component emits a viewchange custom event on the target element for downstream components to subscribe.
