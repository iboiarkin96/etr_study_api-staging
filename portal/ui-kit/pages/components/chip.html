Filter toolbar pattern

When filter chips sit above a card grid or view container, wrap them in the standard .cockpit-toolbar > .cockpit-filter-row pair. Reuse — do not re-implement the spacing:

  • .cockpit-toolbar — outer flex shell with justify-content: space-between and a baked-in margin-bottom: var(--space-4). This is what creates the breathing room between the filter row and the cards below. Drop a view-switcher in here as the second child if you have one; the toolbar handles the gap.
  • .cockpit-filter-row — flex container for the chips themselves (gap: var(--space-2), wraps).
  • Each chip is a <button class="docs-chip docs-chip--filter" data-filter="…">. The outer row carries data-filter-group and data-target="#…" so the auto-mounted filter-chips.js wires click + URL-hash + data-filter-hidden on the items.

Canonical example: ops-cockpit template. The classes live in components/cockpit.css and are intentionally generic — the cockpit- prefix is historical; any page (Tutorials hub, hub landings, dashboards) reuses them as-is.

<div class="cockpit-toolbar">
<div class="cockpit-filter-row"
role="toolbar" aria-label="Filter by role"
data-filter-group
data-target="#my-grid">
<button class="docs-chip docs-chip--filter" data-filter="all"
aria-pressed="true">All</button>
<button class="docs-chip docs-chip--filter" data-filter="dev"
aria-pressed="false">Dev</button>
<button class="docs-chip docs-chip--filter" data-filter="qa"
aria-pressed="false">QA</button>
</div>
<!-- optional second child: view switcher, sort, etc. -->
</div>

<div class="docs-card-grid" id="my-grid">
<a class="docs-card" data-tags="dev" href="…">…</a>
<a class="docs-card" data-tags="qa"  href="…">…</a>
      </div>

States

  • Default — neutral background, 1 px --line border.
  • Hover — background --bg-mute.
  • Active / pressed — accent fill, white text, modifier .docs-chip--active.
  • Disabled — opacity 0.5, no pointer events.

A11y

  • Filter chip is a <button> with aria-pressed.
  • Removable chip's close button must have aria-label="Remove <value>".
  • Group of filter chips: wrap in a container with role="group" and an aria-label.
  • Tap target ≥ 32 × 32 px (small) or 40 px (default).

Tokens used

  • --bg-elev, --bg-mute, --line
  • --accent (active fill)
  • --radius-full
  • --fs-200, --fw-medium
