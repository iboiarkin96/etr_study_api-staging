Reading progress

Fixed 3 px gradient bar pinned to the top of the viewport that tracks how far down the page the reader has scrolled. Auto-injected on every doc page by the kit's runtime — no markup needed.

Example

Look at the very top of this viewport — the thin accent → success gradient is the reading-progress bar tracking this page. Scroll up and down to watch it fill / drain. On short landing pages the bar hides itself (see States).

Static preview of the bar at 42 % scroll — the real bar is 3 px tall and lives at position: fixed; top: 0.

Auto-injection

The kit runtime calls mountReadingProgress(document) on boot. It:

  1. Bails out if a .docs-reading-progress element already exists (idempotent — safe to call twice).
  2. Creates one <div class="docs-reading-progress is-hidden"> with a single child <span class="docs-reading-progress__fill">, and appends it to <body>.
  3. Wires scroll + resize listeners (passive), RAF-throttled.
  4. On every tick, computes scrollTop / (scrollHeight - innerHeight) and applies it as transform: scaleX(ratio) on the fill.

There is no public JS API beyond the mount. No options object, no data-* opt-in / opt-out. To opt out on a specific page, you'd need to remove the runtime entry script — which is not a supported flow.

Markup (injected)

You should not author this markup directly — included for reference / debugging.

<div class="docs-reading-progress"
     role="progressbar"
     aria-label="Page reading progress"
     aria-valuemin="0"
     aria-valuemax="100"
     aria-valuenow="42">
<span class="docs-reading-progress__fill"
        aria-hidden="true"
        style="transform: scaleX(0.42)"></span>
      </div>

States

  • Hidden — .is-hidden (opacity 0) when scrollHeight - innerHeight < 8 px. Pages that fit in the viewport (404, short landings) don't show an empty bar.
  • Tracking — fill scaleX ratio updates each scroll frame; aria-valuenow mirrors the integer percentage.
  • z-index 60 — sits above topbar (50) but below modals / drawer overlays (80+).

A11y

  • role="progressbar" with aria-valuemin="0", aria-valuemax="100", and a live aria-valuenow. Screen readers can query it but most users will skip it (it's decorative reading aid).
  • pointer-events: none — the bar cannot block clicks on the topbar / first row of content below it.
  • aria-label="Page reading progress" gives the role a human name.
  • The inner fill is aria-hidden — only the outer progressbar role is exposed.
  • prefers-reduced-motion: reduce removes the opacity fade transition (the bar still fills, but without the show/hide animation).

Tokens used

  • --line (track background, mixed 66 % with transparent)
  • --accent, --info, --success (fill gradient stops)
  • --ease-out (opacity transition)
  • Fixed values — height: 3px, z-index: 60 (no token; intentional)
