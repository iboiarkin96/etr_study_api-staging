References grid

Two-column references / further-reading block. Each column has a mono uppercase heading and a list of arrow-prefixed links. Use to close out a long-form article with «industry» on one side and «inside this portal» on the other — but two arbitrary buckets (papers / videos, internal / external) work just as well.

Example

Industry

  • Stripe — API versioning How the team carries 12 years of versions in production.
  • Netflix — Chaos Monkey at 10 A decade of failure injection lessons.
  • Spotify — Backstage paper The original «software catalog» argument.

Inside this portal

  • ADR 0007 — OpenAPI governance The contract that keeps our SDKs in sync.
  • RFC 0003 — Documentation authoring How this page itself came to exist.
  • Catalog · Spine matrix What each kind owes; the audit rubric.

Anatomy

  • .pa-refs — grid wrapper, single column on mobile, two columns at ≥760 px.
  • .pa-refs__column — one bucket; contains an <h3> and a flat <ul>.
  • .pa-refs__column h3 — mono uppercase heading, muted (no margin from prose).
  • List items — no bullet; an arrow («→») is rendered via ::before in the tone colour.
  • .pa-refs__hint — small muted block underneath the link, one-line description.

Markup

<div class="pa-refs">
  <div class="pa-refs__column">
    <h3>Industry</h3>
    <ul>
      <li>
        <a href="…" rel="noopener">Title</a>
        <span class="pa-refs__hint">…</span>
      </li>
    </ul>
  </div>
  <div class="pa-refs__column">
    <h3>Inside this portal</h3>
    <ul>…</ul>
  </div>
      </div>

Naming note

Class names use the legacy .pa-refs prefix; neutral rename to .references-grid queued.
