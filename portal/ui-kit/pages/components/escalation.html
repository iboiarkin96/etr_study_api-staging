Escalation

Horizontal pipeline of stages joined by edge labels — used when three stages connect in a meaningful sequence. The canonical case is an on-call handoff (Primary → Secondary → Incident commander); the same shape is reused for any 3-step causal chain where each step activates the next, with a short label on the arrow.

Example

  1. Primary On-call

    Receives the page first.

    • Owns triage, mitigation, communication.
    • Records the reason if a known false-positive is silenced.
  2. no ack ≤ 10 min
  3. Secondary On-call

    Backup — paged automatically.

    • Takes over when the primary does not acknowledge.
    • Covers planned absences for the primary.
  4. incident declared
  5. IC Incident commander

    Coordinates a multi-person response.

    • Worn as a separate “hat” during real incidents.
    • May be the primary, or someone else if the primary is fixing.

Role tones

The data-role attribute on each __stage swaps the per-card accent (top stripe + duty bullets). The progression accent → info → rose reads as “surface to depth”, which fits both the on-call urgency ladder and the trigger/cause/factors causal chain.

  • data-role="primary" — --accent: the default, calmest stage. On-call: Primary on-call. Debugging: Trigger.
  • data-role="secondary" — --info: the middle stage. On-call: Secondary on-call. Debugging: Cause.
  • data-role="ic" — --rose: the deepest / most urgent stage. On-call: Incident commander. Debugging: Contributing factors.

Edge connectors render a gradient line from --accent → --rose with a right-pointing arrowhead. The arrow is drawn via .docs-escalation__edge-arrow (line as background, head as ::after) — no SVG required.

Alternate uses

The component is intentionally generic — three boxes joined by labelled arrows is a versatile shape. Edge labels are the part that changes between uses, not the stages themselves.

  • On-call escalation. Labels: no ack ≤ 10 min, incident declared. Consumer: SRE · On-call.
  • Causal chain. Labels: activates, enabled by. Consumer: SRE · Debugging (Trigger → Cause → Contributing factors).

Pick edge labels that read as a single phrase when joined with the surrounding stages — “Primary no ack ≤ 10 min Secondary” and “Trigger activates Cause” are short enough to scan without re-parsing.

Responsive behaviour

  • ≥1025 px — horizontal 5-column grid (stage / edge / stage / edge / stage).
  • 761–1024 px — same grid, tighter gaps and smaller edge labels.
  • ≤760 px — single column. Each edge becomes a row: short vertical line + down-arrow on the left, label on the right.

A11y

  • The pipeline is an <ol> with an aria-label describing the sequence.
  • Edges are decorative and use aria-hidden="true" — the trigger label is read by sighted users only.
  • Duty lists are real <ul> trees inside each stage; screen readers announce the role pill, stage name, mission, and duties in order.
  • Reduced motion: no transitions or animations are used by this component.

Tokens used

  • --accent, --info, --rose — per-role tones (top stripe, bullets, arrowhead).
  • --card, --line, --bg-elev — stage surface, borders.
  • --text-strong, --text, --muted, --muted-soft — stage name, duties, mission, edge label.
  • --shadow-sm — stage card lift.
  • --radius-lg, --radius-full — card corner, bullet dot.
  • --fs-100 … --fs-500; --fw-medium, --fw-semibold.
  • --font-mono — uppercase edge labels.
  • --space-2 … --space-5.
