Test pyramid

The canonical test pyramid — five stacked CSS clip-path trapezoids narrowing toward the apex, paired with a right-side annotation column whose rows line up with the tiers. The shape itself carries the meaning: wide base = many cheap tests, narrow apex = few expensive ones.

Geometry

Each tier is a clip-path: polygon(…) trapezoid whose top and bottom widths are driven by two unitless CSS variables, --t-top and --t-bot (0..1 fractions of the column width). The default ladder maps tier numbers to widths so the apex sits at 25% and the base at 100%:

  • data-tier="1" — top 0.25, bottom 0.40 (apex).
  • data-tier="2" — top 0.40, bottom 0.55.
  • data-tier="3" — top 0.55, bottom 0.70.
  • data-tier="4" — top 0.70, bottom 0.85.
  • data-tier="5" — top 0.85, bottom 1.00 (base).

Override per-tier inline if a different ladder is needed: style="--t-top: .35; --t-bot: .55". The notes column matches by data-tier attribute (used to pick the tone), and both stacks share a 60 px row height so rows line up vertically without manual offset.

Tones

data-tone swaps the per-tier accent on both the trapezoid background and the note's left border. The defaults read “rare/expensive at the warm top” to “frequent/cheap at the cool base”.

  • data-tone="rose" — --rose (apex, manual/exploratory).
  • data-tone="warn" — --warn (E2E).
  • data-tone="accent" — --accent (contract).
  • data-tone="info" — --info (integration).
  • data-tone="teal" — --teal (base, unit).

Responsive behaviour

  • ≥1025 px — two columns (pyramid 1.4fr, notes 1fr). Tier height 60 px.
  • 761–1024 px — same two columns, gap tightens to --space-4; pyramid takes a slightly larger share (1.2fr / 1fr).
  • ≤760 px — single column. Pyramid sits on top, notes stack below as full-width rows. Tier height drops to 52 px so the apex remains tappable.

A11y

  • The outer container takes role="figure" with an aria-label summarising the pyramid.
  • The tier stack is an <ol> — order matters and is announced by screen readers; the apex is item 1.
  • The notes column is a sibling <ul> — paired by tier number rather than DOM nesting, so each list reads cleanly on its own.
  • Clip-paths do not affect hit-test or accessibility tree; tier labels and notes remain selectable and readable in the order they appear in the DOM.
  • Component-scoped selectors raise specificity so prose.css defaults (ol/ul left padding, bottom margins, li + li top margins) lose the cascade fight.

Tokens used

  • --rose, --warn, --accent, --info, --teal — per-tier tones (gradient fill, note border).
  • --bg-elev, --bg-mute, --line — surfaces, note backgrounds, code chip.
  • --text-strong, --text, --muted — tier label, note title, note rule.
  • --radius-sm, --radius-xs — note corner, code chip.
  • --fs-200, --fs-300; --fw-semibold.
  • --space-3 … --space-5.
  • --ease-out — hover transition.
