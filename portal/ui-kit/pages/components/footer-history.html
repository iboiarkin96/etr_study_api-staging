Footer history

Change-log footer rendered at the bottom of every doc page. Three columns per row: date, author (rendered as a small inline chip), note. The chip falls back to a plain @handle text node when JS is off.

Example

History

  • 2026-05-11 @iboiarkin96 Split internal layout CSS; added analysis & maturity sections.
  • 2026-05-08 @kirill ADR 0030 published — portal shell token contract.
  • 2026-05-06 @cursor Unified prose column width across portals at ≥1025 px.

Markup

<footer class="docs-history" aria-label="History">
<h2 class="docs-history__title">History</h2>
<ul class="docs-history__list">
<li class="docs-history__row">
<span class="docs-history__date">2026-05-11</span>
<a class="docs-history__author"
data-component="author-chip"
data-person-id="16fc8b78537109162984a2fdbef6e142"
data-variant="sm">@iboiarkin96</a>
<span class="docs-history__note">Initial publish.</span>
</li>
</ul>
      </footer>

Author chip auto-mounts on every data-component="author-chip" via mountAuthorChip() in the entry bundle. Anchor inner text is the no-JS fallback handle.

States

Footer history is static — no interactive states.

A11y

  • Use a real <footer> element with aria-label="History".
  • Title is an <h2> so it appears in the page outline.
  • Dates should be ISO-8601 (YYYY-MM-DD); wrap in <time datetime="…"> if you need machine-readable.

Tokens used

  • --muted (date), --muted-soft (author)
  • --line-soft (row separator)
  • --fs-200, --fs-300
  • --space-3
