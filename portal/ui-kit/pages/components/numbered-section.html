SA section

Numbered section frame — small mono "01"/"02"/… kicker badge next to an h2, optional lede paragraph below. Repeats down a long reference page to give it a scannable spine.

Example

01

Anatomy — the universal shape

Every entity card — System, Component, Library, API, Resource — follows the same five-block skeleton.

02

Lifecycle pills — the four states

The first pill in every card is the lifecycle state. Colour is mechanical — readers learn it once and recognise it on every service.

Anatomy

  • .numbered-section — top margin (--space-7) to separate from the previous section.
  • .numbered-section__head — flex baseline row: kicker on the left, h2 on the right, both wrap on narrow viewports.
  • .numbered-section__kicker — mono, uppercased, --muted. Always 2 digits (01, 02, 03.1, …).
  • .numbered-section h2 — margin reset so kicker + h2 sit on the same baseline; tag-default size + weight from prose.
  • .numbered-section__lede — 70ch max-width, muted; one short paragraph that frames what the section covers.

Sub-section numbering

Within a section that needs sub-numbering (e.g. five kinds inside § 02), repeat .numbered-section__head with 02.1, 02.2, … kickers, and use <h3> instead of <h2> so the heading hierarchy stays linear. Example in entity-card.html and spine-matrix.html.

Markup

<section class="numbered-section" aria-labelledby="anatomy-h">
  <div class="numbered-section__head">
    <span class="numbered-section__kicker">01</span>
    <h2 id="anatomy-h">Anatomy — the universal shape</h2>
  </div>
  <p class="numbered-section__lede">…</p>
  <!-- section body -->
      </section>

Tokens used

  • Layout: --space-3 / -5 / -7.
  • Typography: --fs-100, --font-mono, letter-spacing 0.08em.
  • Colour: --muted.
