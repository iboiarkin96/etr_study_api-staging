SA hero

Page-hero primitive for foundations / reference landings — eyebrow chip, gradient-accent title, lede, optional CTAs. Slim sibling of .home-hero with no portrait / no tickers / no canvas. .page-hero__h1 declares its own size + weight so the visual stays consistent on any host tag.

The demo wraps the title in a <p class="page-hero__h1"> so this showcase keeps one true h1. On a real page, use <h1 class="page-hero__h1">.

Anatomy

  • .page-hero — top padding, hairline divider underneath.
  • .page-hero__eyebrow — mono-uppercase line with a pulsing accent dot before it.
  • .page-hero__h1 — fluid clamp(2rem, 4.2vw, 2.8rem), --fw-bold, tight tracking. Wrap any sub-span in .accent for a gradient text effect.
  • .page-hero__lede — 64ch max-width, base body size.
  • .page-hero__ctas — flex row of .docs-btn; stacks vertically on mobile.

Markup

<header class="page-hero">
  <p class="page-hero__eyebrow">SA reference · foundations</p>
  <h1 class="page-hero__h1">Authoring the <span class="accent">SA spine</span></h1>
  <p class="page-hero__lede">…</p>
  <div class="page-hero__ctas">
    <a class="docs-btn docs-btn--primary" href="…">Start authoring</a>
    <a class="docs-btn" href="…">Read principles</a>
  </div>
      </header>

A11y & responsive

  • One <h1> per page — the hero owns it. If you need the same visual elsewhere, use a different tag plus .page-hero__h1.
  • Below 760 px the CTA row stacks vertically and each button stretches full-width.
  • The accent dot in the eyebrow is decorative (CSS-generated); screen readers ignore it.

Tokens used

  • Layout: --space-3 / -5 / -6, --line-soft.
  • Typography: --fs-100 / -400, --fw-bold, --font-mono, --lh-base.
  • Colour: --muted, --text, --text-strong, --accent, --info.
