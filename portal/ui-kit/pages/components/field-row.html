Field row

Enhanced table rows for API field / schema documentation. Adds a Required/Optional dot indicator, a monospace type badge, and an optional Constraints column. Apply .docs-table--fields to the <table> and add data-required to each <tr>.

Example

Sessions · Fields table

Field Type Required Constraints Description
id uuid ● server-assigned Unique identifier.
started_at timestamp ● ISO-8601, UTC Server clock; set on create.
ended_at timestamp ○ ISO-8601, UTC Null while the session is open.
duration_ms int ○ ≥ 0 Derived; only set when ended_at is.
tags string[] ○ ≤ 16 chars each Free-form labels.

Elements

Class / AttributePurpose
.docs-table--fieldsTable modifier — enables data-required row tinting
data-required="true" on <tr>Subtle danger-tinted row background
.docs-field-dot--requiredFilled red circle ●, danger palette
.docs-field-dot--optionalOutlined circle ○, muted palette
.docs-field-typeMonospace type badge, info-tinted
.docs-field-constraintsMonospace constraints cell; renders — when empty

Accessibility

Each dot has title and aria-label so screen readers announce "Required" or "Optional" rather than the unicode bullet character. The column heading "Required" also provides context for sighted users. The type badge is purely decorative monospace text with no additional ARIA needed.

Tokens used

--danger, --info, --muted, --muted-soft, --line, --radius-sm, --radius-full, --font-mono, --fs-100, --fw-medium
