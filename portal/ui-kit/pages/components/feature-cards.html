SA cards

Feature-card grid with per-tile tone — heavier sibling of .docs-card-grid. Each tile gets a radial-gradient wash and a 3-px top accent bar driven by data-tone. Designed for the «four pillars» pattern that opens a foundations landing.

Tones

data-tone="accent | info | teal | rose" drives --card-tone, which colours: the icon background tint, the hint chip, the gradient wash (::before), the top accent bar (::after), the CTA text, and the hover border/shadow.

Markup

<div class="feature-cards">
  <a class="feature-card" href="…" data-tone="accent">
    <div class="feature-card__head">
      <span class="feature-card__icon" aria-hidden="true"><svg …/></span>
      <span class="feature-card__hint">01 · WHY</span>
    </div>
    <h3 class="feature-card__title">Principles</h3>
    <p class="feature-card__body">…</p>
    <span class="feature-card__cta">Open principles <svg …/></span>
  </a>
  <!-- repeat for the other three tiles -->
      </div>

States

  • Hover — translateY(-3px), tone-tinted shadow, gradient wash opacity → 1, border tints toward tone, CTA arrow nudges 3px right.
  • Focus — accent outline (kit default for a:focus-visible).

Responsive

  • ≥761 px — fixed 2-column grid.
  • ≤760 px — collapses to single column.

Tokens used

  • Layout: --space-3 / -4 / -5, --radius-md, --radius-lg.
  • Surface: --bg-elev, --bg-mute, --line.
  • Tone: --accent, --info, --teal, --rose via --card-tone.
  • Typography: --fs-100 / -300 / -500, --font-mono, --fw-semibold, --lh-base.
  • Motion: --ease-out.
