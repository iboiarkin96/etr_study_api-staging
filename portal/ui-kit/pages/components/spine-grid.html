Spine grid

Per-service spine grid that sits below the entity-card on every services/<name>/index.html page. Lists the canonical RFC 0004 §4.4 children — Architecture, API reference, Runbooks, Dependencies, On-call, Code reference — as a responsive grid of tone-tinted anchor tiles that share the entity card's design language (dashed border, corner halo, accent rail, mono index, hover lift).

Tones

Selector inventory

  • .spine-grid[data-spine-tone="…"] — outer wrapper. Renders as a CSS grid with repeat(auto-fill, minmax(280px, 1fr)); tone exposed as --spine-tone.
  • .spine-tile — anchor tile with dashed/solid tone border, corner halo (::before), 2 px accent rail on the top edge (::after), hover lift + glow.
  • .spine-tile__head — flex row with the index on the left and the icon chip on the right.
  • .spine-tile__index — mono "01 · Architecture" label, tone-tinted to color-mix(--spine-tone 70 %, --muted).
  • .spine-tile__icon — 36×36 tone-chip with 10 px radius; tints harder on hover via --spine-tone.
  • .spine-tile__title — mono --fs-500 heading.
  • .spine-tile__body — one-paragraph summary (--fs-300 on --muted).
  • .spine-tile__foot — dashed top hairline (color-mix(--spine-tone 22 %, --line-soft)), CTA on the left, arrow on the right.
  • .spine-tile__cta — uppercase mono "Open" label tinted toward the tone.
  • .spine-tile__arrow — translates 3px right on hover / focus-visible.

States

  • Default — 1 px tone-tinted border, soft halo at 85 % opacity, 2 px tone rail on the top edge.
  • Hover / focus-visible — lift translateY(-2px), dual-layer tone glow, halo to 100 %, icon chip background pumps from 14 % → 22 % tone wash; arrow slides 3 px right.
  • Focus-visible — additional 2 px outline tinted at 60 % tone with 3 px offset.
  • Mobile (≤ 760 px) — grid collapses to a single column with --space-3 gap; tile padding tightens.

A11y

  • Each tile must be a single <a> — the whole surface is the target. Do not nest a button inside.
  • Render the grid as a <ul> with <li> children so screen readers announce it as a list.
  • Decorative icons and the arrow live inside aria-hidden spans; the index "01 · Architecture" carries the meaning in text.
  • Halo and accent rail are pointer-events: none and never trap focus.
  • Focus ring is tone-tinted but always ≥ 3:1 contrast against the surface — falls back to the kit's global ring under forced-colors.

Tokens used

  • --spine-tone (local, set by data-spine-tone) ← --accent (default) / --info / --success / --warn / --danger
  • --bg-elev, --line, --line-soft
  • --text, --text-strong, --muted
  • --font-mono, --fs-300, --fs-400, --fs-500, --fw-semibold
  • --space-3, --space-4, --space-5, --space-6
  • --radius-lg, --ease-out (hover lift + arrow slide)
