Your first PR through the gate

Make one trivial change, run every check that CI will run, push the branch, and open the PR. The goal is to feel the gate — pre-commit, make verify, PR body conventions — not to ship anything meaningful. Real feature work just adds substance on top of this same shape.

⚙️ 4 phases ⏱ 20–30 min 📚 First-time 💻 Local stack

– Before you start

  • Repo cloned, on main, git status clean.
  • make venv && make install already ran successfully once.
  • Push access to the repo (or a fork). GitHub CLI (gh) optional but recommended.

0 Goal

  • A branch tutorial/first-pr-<your-handle> with one tiny change.
  • Pre-commit ran on staged files and passed.
  • make verify green locally — same checks CI will run.
  • A PR opened against main, body following the template.

1 Branch and pick a trivial change

Per ADR 0017 (branch naming), tutorial branches use the tutorial/ prefix: 

git switch -c tutorial/first-pr-$(whoami)

For the change itself, pick the smallest thing in the repo that nobody else will notice: a typo in a docstring you spotted, a missing trailing newline, or one extra word in a CHANGELOG-style file. If nothing comes to mind, add your name as a maintainer chip on the tutorials hub's history note — single-line edit.

✓ Check your progress

git status shows exactly one modified file. git diff --stat shows one or two lines changed. If the diff is larger, narrow the change — a tutorial PR is not a refactor.

2 Run pre-commit on the staged change

Install the hooks once per clone (idempotent):

make pre-commit-install

Stage and run the hooks against just your file:

git add <your-file>
make pre-commit-check

Hooks cover formatting, lint, dead-code, secret scan, and a docs sanity pass. Anything that auto-fixes will modify your working tree — re-stage and re-run until the line ends with pre-commit: OK.

✓ Check your progress

Last log line reads pre-commit: OK. If a hook keeps reformatting the same line, your editor is fighting it — disable format-on-save for this file or accept the hook's version.

3 Run the full local gate

make verify is the same command CI runs. It is the single source of truth for "PR is mergeable": 

make verify

It chains: format check, lint, type-check, dead-code check, the test suite, OpenAPI contract, docs HTML/design/a11y, and dependency audit (see ADR 0008 for the full taxonomy). Last log line should read verify: OK.

If verify fails, the failed step is named on the last error line. Most first-PR failures are pre-commit drift (run make fix and retry) or a stale snapshot (make docs-fix for docs, or make openapi-accept-changes if you intentionally changed an endpoint).

✓ Check your progress

make verify ends with verify: OK. Anything else, do not push — the same failure will hit CI in 30 seconds and waste a slot.

4 Commit, push, and open the PR

Commit with a conventional message (one-liner is fine for a tutorial change):

git commit -m "chore(tutorial): first-PR walkthrough by $(whoami)"
git push -u origin HEAD

Open the PR. With gh:

gh pr create --base main --title "chore(tutorial): first PR walkthrough" --body "$(cat <<'EOF'
## Summary
- Walking through the first-PR tutorial.

## Test plan
- [x] `make pre-commit-check` green
- [x] `make verify` green locally

EOF
)"

Without gh, the push output prints a "create pull request" URL — open it, paste the same body manually.

Open the PR in the browser (gh pr view --web if you used the CLI, or click the URL the push printed). Confirm the title, body, and base branch look right, and that a CI run has appeared within ~1 minute.

✓ You're done

  • Branch tutorial/first-pr-<your-handle> pushed to the remote.
  • PR open against main with the template body, visible in the browser.
  • CI started within a minute and is running the same make verify you ran locally.

That is the contribution loop in this repo: branch → pre-commit → make verify → PR. Real feature work just adds substance on top of this same shape.

↺ Want to start over (or drop this PR)?

gh pr close --delete-branch  # if a PR was opened
git switch main
git branch -D tutorial/first-pr-$(whoami) 2>/dev/null || true

gh pr close --delete-branch drops the PR and the remote branch in one shot. The local branch is removed by the last line.
