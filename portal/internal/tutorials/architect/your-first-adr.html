Tutorial Architect · Foundations

Your first ADR, end-to-end

Capture a trivial decision as a numbered ADR, register it in the index and the sidebar, and watch it appear in portal search. The point is the governance loop — copy template → fill → number → register → regenerate — not to ship policy. Real ADRs use the same shape with substantive content.

⚙️ 4 phases ⏱ 25–35 min 📚 First-time 💻 Local stack

– Before you start

  • Portal serves locally (see local development).
  • GNU make, Python 3.11+, git.
  • You've read ADR 0018 — ADR lifecycle once (one page); it defines the states your tutorial ADR will move through.

0 Goal

  • One new file services/portal/internal/governance/adr/XXXX-tutorial-sample.html (where XXXX is the next free number).
  • The ADR linked from the ADR index page.
  • The ADR visible in the sidebar under Governance → ADR.
  • Portal search returns it for the keyword tutorial sample.

The decision content is intentionally trivial ("we will end every tutorial page with a History footer") — production ADRs follow the same shape with real context, options, and consequences. The substance is in the decision-records practice; this tutorial only teaches the file-and-publish loop.

1 Pick the next number and copy the template

List the existing ADRs and find the highest number:

ls services/portal/internal/governance/adr/ | grep -E '^[0-9]{4}-' | sort | tail -3

Your number is highest + 1, four-digit, zero-padded. Throughout the rest of this tutorial we'll call it XXXX; substitute your actual number every time.

Copy the template:

cp services/portal/internal/handbook/sa/templates/adr.html \
        services/portal/internal/governance/adr/XXXX-tutorial-sample.html

✓ Check your progress

git status shows one new untracked file with your chosen number. Open it in the browser at http://127.0.0.1:3000/services/portal/internal/governance/adr/XXXX-tutorial-sample.html — it renders as the template (placeholder text, status pills).

2 Fill in the minimum fields

Edit the new file. The template has many sections; for this tutorial only touch these four — leave the rest as-is or delete sections you don't need:

  1. <title> and <h1>: replace placeholder with ADR XXXX — End every tutorial with a History footer.
  2. Context: one sentence. Example: Tutorials drift quickly; future-me needs to see who changed what and when.
  3. Decision: one sentence. Example: Every tutorial page ends with a <footer class="docs-history"> block.
  4. Status: leave as Proposed — this is the lifecycle entry state per ADR 0018.

✓ Check your progress

Refresh the page in the browser. Title, h1, context, and decision show your edits. Status pill still reads Proposed.

3 Register in the ADR index and the sidebar

Open services/portal/internal/governance/adr/index.html. Find the most recent ADR row in the listing and add a sibling for your ADR using the same markup pattern (date today, status Proposed, link to your file).

Then open services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.json and locate the existing entry for the latest ADR — copy it as a sibling right below, with your number and slug: 

{
"id": "governance-adr-XXXX-tutorial-sample",
"label": "XXXX — Tutorial sample",
"href": "/services/portal/internal/governance/adr/XXXX-tutorial-sample.html"
}

Validate the JSON before regenerating:

python -m json.tool < services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.json > /dev/null

✓ Check your progress

JSON validation is silent (no output). The ADR index page lists your ADR in its rightful place. The sidebar — once we regenerate in Phase 4 — will show it under Governance → ADR.

4 Regenerate and verify 

make docs-fix

This rebuilds the search index and the static sidebar mirror. Last log line should read docs-fix: OK.

✓ You're done

  1. Sidebar (any internal page): Governance → ADR → XXXX — Tutorial sample.
  2. ADR index page lists the new entry in Proposed.
  3. Portal search finds the page when you type tutorial sample.

That is the ADR publishing loop — pick number → copy template → fill → register × 2 → make docs-fix. Production ADRs follow the same shape; the difference is what you write into Context, Decision, and Consequences.

↺ Want to start over?

rm services/portal/internal/governance/adr/XXXX-tutorial-sample.html
git checkout -- services/portal/internal/governance/adr/index.html \
services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.json
make docs-fix
