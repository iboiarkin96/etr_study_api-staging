History format

Every hand-written page ends with a History footer. One row per substantive edit. Skip typo-only changes. This is the audit trail without leaving the page.

Markup

<footer class="docs-history" aria-label="History">
  <h2 class="docs-history__title">History</h2>
  <ul class="docs-history__list">
    <li class="docs-history__row">
      <span class="docs-history__date">2026-05-18</span>
      <a class="docs-history__author"
         data-component="author-chip"
         data-person-id="16fc8b78537109162984a2fdbef6e142"
         data-variant="sm">@ivan</a>
      <span class="docs-history__note">One-line note describing the substantive change.</span>
    </li>
  </ul>
      </footer>

Fields

The cell vocabulary — three required fields per row, plus the page-level maintainer attribute.

  • docs-history__date — ISO YYYY-MM-DD. The calendar date of the edit, not the file's last-modified timestamp. One date per row; never a range.
  • docs-history__author — an <a> wrapped with data-component="author-chip". The runtime hydrates the chip (avatar + name) from data-person-id, which holds the maintainer UUID from team/people/. Inner text is the display handle (e.g. @ivan); for legacy rows where the author is unknown, render «—» as plain text.
  • docs-history__note — one present-tense sentence describing what changed, not who changed it or why the page exists. Use <code> spans for symbol, class, file, or anchor names. Empty or filler text («updated», «minor changes») is not legal.
  • data-maintainer-ids (on <body>) — space-separated maintainer UUIDs for the page. Not a row field — a page-level attribute that lists every current maintainer.

Order

The <ul class="docs-history__list"> reads top-down as a reverse-chronological log: newest row at index 0, older rows below. Rows are immutable once published — never reorder, never backdate, never delete.

Examples

Legal note — substantive change, present tense, <code> spans for class names:

<span class="docs-history__note">Migrated to UI Kit v3: <code>docs-prose</code> + <code>page-head</code>; sections become h2 with id anchors.</span>

Illegal note — empty filler, no information about what changed:

<span class="docs-history__note">Updated.</span>
