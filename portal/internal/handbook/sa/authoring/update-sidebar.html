Update the sidebar

A new portal page is invisible until the sidebar manifest lists it. This guide walks the edit, the validation step, and the recovery path when something goes wrong. The schema lives once — on the nav-tree schema reference — and is not restated here.

Before you start

  • You have a new portal page committed to services/portal/internal/… and you know its full href.
  • You have skimmed the nav-tree schema reference at least once so the field names below are not a surprise.
  • You have python3 on your PATH (used by the validation step).
  • The repo is on the branch where the page itself lives — sidebar edits travel in the same PR as the page they enable.

Steps

  1. Decide where the new entry goes. Match the document's kind against the placement map: API spec → entity hub's Operations group, ADR → topical ADR cluster, runbook → Runbooks, tutorial → role group, and so on. Note the parent group's id — you will paste the new entry inside its children.

  2. Add the JSON entry. Open services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.json, find the parent group, and append a leaf:

    {
  "id": "foundations-sa-templates-adr",
  "label": "ADR",
  "href": "/services/portal/internal/foundations/reference/sa/templates/adr.html"
          }

    Preserve acronym casing (API, AsyncAPI, SRE, gRPC, REST, SQL, ADR, RFC, QA, CI). Watch the trailing comma — JSON does not forgive one after the last array element.

  3. Lint the JSON manifest locally. Trailing-comma and brace-balance bugs are the most common breakage:

    python3 -c "import json; json.load(open('services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.json'))"

    Silence means success.

  4. Run the full docs gate. make docs-check covers asset-ref validation and search-index drift — both of which can break after a sidebar edit.

Verify

  • JSON parses. The lint command above exits 0 with no output.
  • New entry appears in the sidebar. Open the new page in localhost:3000 and confirm the label is present and active in the left rail.
  • Acronyms render correctly. Spot-check the label text — drift here is the most common navigation bug in the portal.
  • Asset-ref check passes. Pre-commit hook check-asset-refs reports zero broken references for the manifest changes.
  • No duplicate id. Search the JSON for the new ID — exactly one match.

Troubleshooting

  • The sidebar is blank after my edit. JSON parse failed silently. Re-run the json.tool lint command from step 3 — it will point to the line with the trailing comma or unbalanced brace.
  • The new entry shows up, but the link is a 404. The href path does not match the file on disk. Re-check the spelling, the directory casing, and that it starts with /services/portal/internal/ and ends with .html.
  • check-asset-refs hook blocks the commit. The target file does not exist yet (typo or wrong path). Fix the path or create the file before re-running.
  • Duplicate id silently drops both entries. If two entries share an id the sidebar component drops both. Search the JSON for the ID and rename one.
