Service How-to

Run the API locally

From a fresh clone to a running API on http://127.0.0.1:8000, plus the optional observability and logging stacks. The fallback page when setup hits friction.

Prerequisites

  • Python 3.14 on the system PATH (see .python-version for the exact pin). Older minors will install but break some scripts.
  • GNU Make (macOS ships it; on Linux it is in the distro tooling).
  • Git, with the repo cloned.
  • Docker is optional. You only need it for the observability stack (Prometheus + Grafana) or the logging stack (Elasticsearch + Kibana + Filebeat). The API itself runs without Docker — see Build and run as Docker image for when you do need it.

First-time setup

One command from a fresh clone:

make setup

That target chains three steps so a fresh checkout is ready in one go:

  1. make venv — create .venv/ via python3 -m venv .venv. Skipped if it already exists.
  2. make install — upgrade pip, then pip install -r requirements.txt into .venv. Every dependency is pinned (==) — both direct and transitive — so the env is reproducible.
  3. make env-init — copy env/example → .env. Skipped if .env already exists.

Open .env and set at least APP_ENV=dev and an API_MOCK_API_KEY (anything is fine in dev). The full env-var contract and load order live in env/example — it is the source of truth.

Run the API

make run

This target reads .env, runs alembic upgrade head against the configured database, then starts Uvicorn with --reload. Defaults: host 127.0.0.1, port 8000; override with APP_HOST / APP_PORT in .env or on the command line.

Quick verification once the server boots:

  • http://127.0.0.1:8000/live — liveness probe (no auth).
  • http://127.0.0.1:8000/ready — readiness probe (DB reachable).
  • http://127.0.0.1:8000/docs — Swagger UI for the OpenAPI surface.
  • http://127.0.0.1:8000/metrics — Prometheus exposition (requires the obs stack to scrape it).

Migrations alone, without starting the server: make migrate.

The daily loop

Most days look like this:

  • Write code. The Uvicorn reload picks up edits under services/api/app/ automatically.
  • Run tests. make test runs the full pytest suite with coverage; make test-one path=tests/api/v1/test_user_create.py isolates one file or node.
  • Auto-fix. make fix runs format-fix + lint-fix + docs-fix in sequence. Run it before make verify to keep the diff clean.
  • Verify. make verify is the strict gate — deps audit, lint, types, OpenAPI, contract test, tests, docs drift, docs a11y. Same gate CI runs.

Observability stack (optional)

For metrics dashboards locally:

docker compose -f services/monitoring/docker-compose.observability.yml up -d
      make run

Prometheus comes up at http://127.0.0.1:9090 and scrapes the API's /metrics endpoint. Grafana at http://127.0.0.1:3001 (default login is documented inside the compose file). The API exposes Prometheus metrics out of the box; no extra config needed.

Tear down: docker compose -f services/monitoring/docker-compose.observability.yml down (add -v to wipe volumes).

Logging stack (optional)

For log search in Kibana locally:

  1. Start the stack: make logging-up (Elasticsearch + Kibana + Filebeat via services/monitoring/docker-compose.logging.yml).
  2. Make sure the API writes NDJSON to ./logs/: LOG_FORMAT=json in .env (this is the default). Filebeat tails LOG_DIR=logs and ships into ES.
  3. Open Kibana at http://127.0.0.1:5601 and create a data view for *study-app-logs* with @timestamp as the time field.
  4. Health check: curl -fsS http://127.0.0.1:9200/_cluster/health for Elasticsearch; docker compose -f services/monitoring/docker-compose.logging.yml ps to confirm Filebeat is running. CLI search by request id: curl -fsS "http://127.0.0.1:9200/study-app-logs-*/_search?q=request_id:<uuid>&pretty" (or use the Kibana UI for richer filters).

Tear down: make logging-down (preserves volumes — restart is fast). To wipe ES volumes too: docker compose -f services/monitoring/docker-compose.logging.yml down -v. Background and rationale: ADR 0023 — structured logging and local Elasticsearch.

Troubleshooting

SymptomWhat to check
make run says .env not foundmake env-init, then edit .env.
make run says .venv not foundmake venv && make install, or just make setup.
Port 8000 already in useStop the other process, or set APP_PORT=8001 in .env.
Migration error on bootmake migrate on its own to see the full Alembic trace; check SQLITE_DB_PATH in .env.
make verify fails on docs driftmake docs-fix, commit the regenerated artifacts, retry.
Pre-commit hook fails on staged filesFix the underlying issue, re-stage, recommit (do not skip with --no-verify).
Kibana shows zero eventsConfirm LOG_FORMAT=json and the API actually wrote to ./logs/app.log; check Filebeat is up via docker compose -f services/monitoring/docker-compose.logging.yml ps.
Grafana dashboards emptyConfirm Prometheus is up at :9090 and that /metrics on the API is reachable from inside the obs container.

Verify

  • make run stays in the foreground without immediate exit; the last log line names the Uvicorn bind http://127.0.0.1:8000.
  • curl -i http://127.0.0.1:8000/live returns HTTP 200 with a JSON body.
  • curl -i http://127.0.0.1:8000/ready returns HTTP 200 after Alembic upgrade head completes (logs visible in the terminal).
  • make verify exits 0 — same checks CI will run, includes lint / type / test / docs gates.
  • No pre-commit drift on a clean tree: git status is empty after make verify.
