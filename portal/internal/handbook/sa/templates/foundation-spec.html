How to use this template

Golden template for any foundation page under services/portal/internal/front/foundations/. A foundation describes a cross-cutting visual or runtime rule used by many components — tokens, typography, motion policy, breakpoints, runtime modules. It is not a single UI part: components reference foundations to stay consistent.

Required vs optional. Required sections must exist and be non-empty. Optional sections may be removed when they do not apply — never reorder, only remove.

If you are documenting a single UI part, use the component spec template instead.

Lint checklist (block-merge gate, future)

This block lives only on the template page. Real foundation specs do not duplicate it — when the front-spec linter is added (Phase 3), it will read data-spec-section / data-spec-required attributes from the page and apply the rules below.

  • Body attributes: data-spec-page="foundation", data-spec-status, data-spec-owner on <body>.
  • Page-type badge in the H1 reads Foundation.
  • Every required section exists as <section data-spec-section="…" data-spec-required="true"> with non-empty content.
  • Tokens / values contains a table or definition list with at least one row.
  • Usage rules contains both when to apply and when not to apply.
  • Examples shows visual or code demonstrations (inline SVG, screenshot, or code block).
  • Code handoff names every implementing file and selector group.
  • History section present with at least one entry.
