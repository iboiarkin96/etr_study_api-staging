Summary

End-user identity for the platform. One row per learner inside a partner integrator's tenant; the composite natural key (system_user_id, system_uuid) uniquely identifies a learner, while client_uuid is the server-issued stable primary key used as the ownership target on every downstream resource.

Schema

ColumnTypeNullableDefaultFKIndexDescription
client_uuidString(36)nouuid4()—PKServer-issued stable identity; referenced as owner_client_uuid by every domain table.
system_user_idString(128)no——uq_users_compositeExternal user identifier from the integrator. Part of the composite natural key.
system_uuidString(36)no—systems.uuiduq_users_compositeExternal tenant UUID. Part of the composite natural key.
full_nameString(256)no———Display name. PII medium — masked at warn level in logs.
usernameString(64)yesNULL——Optional handle. PII medium.
timezoneString(64)yesNULLtimezones.iana—IANA zone (e.g. Europe/Berlin) used to compute calendar-day boundaries server-side.
invalidation_reason_uuidString(36)yesNULLinvalidation_reasons.uuid—Reason this row was soft-invalidated; NULL while valid.
is_row_invalidIntegerno0——0 = valid; 1 = soft-deleted. Rows are never hard-deleted.
created_atDateTimenoNOW()——Insertion timestamp.
updated_atDateTimenoNOW()——Mutation timestamp; bumped on every successful PUT/PATCH.

Constraints

  • Primary key: client_uuid.
  • Unique constraints: uq_users_composite on (system_user_id, system_uuid).
  • Indexes (non-FK): none beyond the composite UNIQUE.

Relationships

  • Belongs to: systems via system_uuid (N-1).
  • Has many: conspectuses via owner_client_uuid (1-N); error_logs via owner_client_uuid (1-N).

Lifecycle

  • Created by: POST /api/v1/user.
  • Updated by: PUT / PATCH /api/v1/user/{system_uuid}/{system_user_id}.
  • Read by: GET /api/v1/user/{system_uuid}/{system_user_id}; ownership checks on every conspectus / error-log endpoint.
  • Retention: soft-delete via is_row_invalid = 1; rows are never hard-deleted (FK targets across the row graph).
