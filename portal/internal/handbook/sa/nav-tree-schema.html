Nav-tree schema

Authoritative shape of the JSON manifest that drives sidebar navigation in the internal portal. This page is the contract. For the procedure of adding a new entry, see the Update sidebar how-to.

The manifest

FileRoleConsumed by
services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.json Sole source of truth for the internal sidebar. Sidebar Web Component ui-kit/components/sidebar.js; pages declare nav source via the data-nav-src attribute on the <aside>.

JSON entry — leaf node

{
  "id": "foundations-sa-templates-adr",
  "label": "ADR",
  "href": "/services/portal/internal/foundations/reference/sa/templates/adr.html"
      }
FieldTypeRequiredRules
idstringyesDot-free, kebab-case, prefixed with the section path. Must be unique within the manifest.
labelstringyesSentence case with acronym preservation (API, AsyncAPI, SRE, gRPC, REST, SQL, ADR, RFC, QA, CI).
hrefstringyes (for leaves)Absolute from the repo root, must start with /services/portal/internal/, must end with .html.
iconstringnoOptional emoji or short label tag rendered before the link.

JSON entry — group node

{
  "id": "foundations-sa-templates",
  "label": "Templates",
  "children": [
    { "id": "foundations-sa-templates-adr", "label": "ADR", "href": "/services/portal/internal/foundations/reference/sa/templates/adr.html" },
    { "id": "foundations-sa-templates-rfc", "label": "RFC", "href": "/services/portal/internal/foundations/reference/sa/templates/rfc.html" }
  ]
      }

A group node has children instead of href; children may themselves be leaves or groups. Nesting is unbounded but the sidebar component visually flattens beyond two levels — keep three-deep as the practical maximum.

Placement map

Where a new entry belongs by document kind. Authors who already know what they wrote can jump straight to the correct subtree without scanning the whole manifest.

Document kindSubtree
API operation specreference → entity hub → Operations
ADRgovernance → ADRs → topical cluster (Documentation, Security, API contract, …)
RFCgovernance → RFCs
Runbookhow-to → Incidents → Runbooks
Postmortemhow-to → Incidents → Postmortems
Tutorialtutorials → role group
How-tohow-to → topic group
Reference page (other)reference → relevant block (api, data, front, qa, manager, uml)
Handbook / Foundationshandbook or foundations → matching sub-group

Invariants enforced by tooling

  • Valid JSON. python3 -m json.tool must parse the manifest. make docs-check runs this implicitly.
  • Unique id. Duplicate IDs cause the sidebar component to drop entries silently.
  • Reachable href. check-asset-refs pre-commit hook rejects entries whose target file is missing.
  • Acronym casing preserved. The reviewer is the enforcer — there is no tool that distinguishes "Adr" from "ADR" automatically.

Anti-patterns

  • Relative href in the manifest. Always absolute (/services/portal/internal/…).
  • Acronym lower-casing ("label": "Adr") — fails review even if CI passes.
  • Mixing href and children on the same node. A node is either a leaf or a group, not both.
  • Trailing commas after the last array / object element — common after a deletion.
