Accessibility (WCAG 2.1 AA)

A reusable template for the WCAG 2.1 AA manual + tool-assisted pass. Instantiate one run per page-under-test or portal feature; walks through the four POUR principles. Procedure lives in the accessibility playbook.

How to use this template

  1. One run per page-under-test or per portal feature (a route, a component, a flow).
  2. Copy this page to audits/<YYYY-MM-DD>-<target-slug>/accessibility.html, or paste the tables into the audit ticket.
  3. Fill the Run block, then walk every POUR table top-to-bottom and fill Status / Evidence / Notes.
  4. Apply the Pass / fail rules and post the verdict.

Preconditions

This checklist assumes the following are in place before any row is scored.

  • A current Chromium-based browser and one WebKit browser are available on the test machine.
  • axe DevTools (or an equivalent rule-set) is installed in the test browser.
  • A screen reader is available — VoiceOver on macOS, NVDA on Windows.
  • The page under test has been loaded with cache disabled (DevTools → Network → Disable cache).

Pass / fail rules

  • Pass: no critical violation; all rows pass; axe scan clean.
  • Fail: any keyboard trap, missing alt on informative image, contrast failure on body text, or missing form label.
  • Needs follow-up: minor issue with a tracked ticket (e.g. one decorative icon missing aria-hidden).
