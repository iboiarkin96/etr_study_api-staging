Reference QA Checklist

Documentation pages — visual verification

Manual visual + interaction pass for any documentation page on desktop and mobile. Use this as a reusable template: instantiate one run per page-under-test (or per portal release), fill the Run block, then mark every check Pass / Fail / N/A with evidence.

How to use this template

  1. Decide your target — a specific page URL, a portal release, or a set of pages.
  2. Copy this page to audits/<YYYY-MM-DD>-<target-slug>/documentation-visual.html, or paste the tables below into the audit / release ticket.
  3. Fill the Run block, then walk every table top-to-bottom and fill Status / Evidence / Notes.
  4. Apply the Pass / fail rules and paste the verdict into the ticket.

Preconditions

The checklist assumes the portal is reachable and the runner has the right browsers available.

  • The docs home page and at least one page from each major section are loaded in a tab.
  • A Chromium-based browser and a WebKit browser are available on the test machine.
  • Light and dark theme modes are both verifiable (theme toggle works).

Pass / fail rules

  • Pass: all rows verified; no blocking UI defects.
  • Fail: any broken navigation, inaccessible control, or severe visual regression.
  • Needs follow-up: minor visual issue that does not block content access.
