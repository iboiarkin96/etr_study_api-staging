1. Overview

Assessment type: UI/UX consistency. Formation date: 2026-06-04.
Snapshot: commit 85c587d (branch main, 2026-06-02 merge).
Previous assessment: UI/UX Assessment — 2026-05-25 (superseded).
Previous DX assessment: DX Assessment — 2026-06-02.

Overall: 6.4 / 10 (−0.3 vs 2026-05-25). The portal's token layer, chrome uniformity, and sidebar are in good shape. The regression comes from two newly confirmed gaps: (1) canonical page templates in assets_v2/ui-kit/templates/ declare classes that zero live pages use — the templates are aspirational rather than normative; (2) three template files in handbook/sa/templates/ contain inline style= attributes that violate the 2026-05-29 no-inline-style rule and will propagate to author copies. Motion duration tokens remain absent. docs-callout adoption is negligible at 3 uses across 464 pages.

2. Scope and methodology

Scope

In scope: all 464 HTML files under services/portal/internal/; design token files at services/frontend/portal/assets_v2/ui-kit/tokens/; component CSS at assets_v2/ui-kit/components/; page templates at assets_v2/ui-kit/templates/; runtime entry points at assets_v2/runtime/internal/.

Out of scope: services/portal/public/ (covered separately); services/portal/internal/team/people/ivan-boyarkin/notes/ (personal scratchpad); product UI outside the docs portal.

Methodology

The audit ran in two parallel sweeps. Sweep A: full-corpus grep for inline styles, data-page-type presence, docs-history presence, asset path patterns, pill class usage, table class usage, and nav-src consistency. Sweep B: selective deep-reads of canonical templates, token files, the runtime entry.css import chain, 3 ADR pages, 4 tutorial articles, 4 section index pages, and 2 how-to article pages — compared against the declared canonical templates. Benchmarks: Stripe Docs and Linear (primary); Vercel and GitHub Primer (secondary).

Confirmed vs not-confirmed

Every finding in Tables 1–2 and Top-7 Gaps is marked confirmed (file:line evidence cited) or not confirmed (observed from prior reports, not re-verified at this snapshot).

What good looks like for a top-decile docs portal UI/UX. Benchmark sources: Stripe Docs, Linear, Vercel, GitHub Primer, Radix Themes, Shopify Polaris, Material 3, Apple HIG.

7. Top-7 gaps — detail

Gap 1 — Canonical templates are aspirational, not normative

Severity: SEV-2. Type: template. Impact × Effort: High impact (authoring confusion for new contributors), low-to-medium effort (decision + search-replace).

Evidence:

  • assets_v2/ui-kit/templates/landing-section.html:20 — body class: docs-section-landing. Used by zero live pages.
  • assets_v2/ui-kit/templates/landing-section.html:32 — hero class: docs-section-hero. Used by zero live pages.
  • assets_v2/ui-kit/templates/doc-howto.html:19 — body class: docs-article docs-article--howto. Used by zero live pages.
  • assets_v2/ui-kit/templates/doc-howto.html:30 — main class: docs-prose docs-prose--howto. Used by zero live pages.
  • 35 section index pages: all use home-hero home-hero--section (confirmed by grep).
  • 29 how-to pages: all use class="docs-shell" + <main class="container"> + <article class="docs-prose">.

Decision required: Option A — update the templates to match the actual live implementation (the live implementation is internally consistent and works). Option B — migrate all live pages to the template's shell classes. Option A is lower risk and lower effort. If Option A is chosen, the UI Kit showcase page for landing-section and doc-howto must also be updated to reflect the real classes.

Owner suggestion: frontend / docs.

Gap 2 — Inline styles in template files

Severity: SEV-2. Type: component, template. Impact × Effort: Medium impact (templates are reference material for authors — violations propagate), low effort (3 files, 4 occurrences).

Evidence (exact file:line):

  • handbook/sa/templates/adr.html:67 — style="vertical-align: middle; font-size: 0.65em;" on a .docs-pill span. Overrides kit-defined pill sizing. Fix: promote to a .docs-pill--sm kit modifier or a .docs-pill--inline utility class.
  • handbook/sa/templates/data-table.html:53 and :114 — style="--svc-tone: var(--neutral);" setting a custom property inline. Fix: use a data-tone attribute handled by the component JS, or a BEM modifier class.
  • handbook/sa/templates/screen-spec.html:321–322 — style="width: 50%;" on two <th> elements. Fix: use class="docs-table__col--half" with the width rule in table.css.
  • Non-template pages: 6 radar pages at team/roles/*/radar.html — style="margin-top:14px" on .read-tip. Fix: promote to radar.css rule.
  • Non-template page: internal/index.html:109 — style="grid-column: span 2;" on an anchor. Fix: add a .docs-card--span-2 modifier to card.css.

Owner suggestion: frontend (kit additions), docs (template file updates).

Gap 3 — Missing docs-history footer on root and radar pages

Severity: SEV-2 for internal/index.html; SEV-3 for radar pages. Type: component. Impact × Effort: Medium impact (provenance/trust signal absent on highest-traffic page), very low effort.

Evidence:

  • services/portal/internal/index.html — no docs-history element found. This is the root landing page — the most visible page in the portal.
  • team/roles/qa/radar.html — no docs-history.
  • team/roles/manager/radar.html — no docs-history.
  • team/roles/sa/radar.html — no docs-history.
  • team/roles/architect/radar.html — no docs-history.
  • team/roles/swe/radar.html — no docs-history.
  • team/roles/sre/radar.html — no docs-history.

Fix: Add <footer class="docs-history" aria-label="History">…</footer> with at least one history entry to each of these 7 pages.

Owner suggestion: docs.

Gap 4 — 36 API code-reference pages missing data-page-type

Severity: SEV-2. Type: template, maintenance. Impact × Effort: Medium impact (search ranking signal and quadrant routing broken for entire API code-reference subtree), medium effort (must be addressed in the pdoc generation pipeline).

Evidence: The entire services/portal/internal/services/api/code-reference/ subtree (36 pages) contains no data-page-type attribute on the <body> element. These are pdoc-generated pages. The search index may classify them under an unknown quadrant or exclude them from filtered views.

Fix: Add data-page-type="reference" to the pdoc HTML template. Additionally, add data-maintainer-ids if absent.

Owner suggestion: frontend (pdoc template).

Gap 5 — Motion duration tokens absent

Severity: SEV-3. Type: token. Impact × Effort: Medium impact (animation timing is not composable; a redesign or A/B test of animation speed requires touching every component file), very low effort (define 3 tokens).

Evidence: _base.css:33–36 defines --ease-out, --ease-in-out, --ease-spring. No --duration-* token defined in any file under assets_v2/ui-kit/tokens/. Component CSS files use raw ms values (not verified per-component but absence from tokens is confirmed).

Fix: Add to _base.css:

--duration-fast:   120ms;
--duration-base:   200ms;
        --duration-slow:   350ms;

Then sweep component CSS to replace raw ms literals. The @media (prefers-reduced-motion) block in _base.css:73 already overrides durations portal-wide via !important, so the token values will be correctly zeroed for reduced-motion users without further changes.

Owner suggestion: frontend.

Gap 6 — docs-callout adoption negligible

Severity: SEV-3. Type: component. Impact × Effort: High impact (content without callouts is harder to scan; authors improvise inline emphasis instead), medium effort (document the pattern, update authoring templates).

Evidence: docs-callout--{tone} found in 3 files out of 464. role-callout role-callout--{type} found in 163 uses — but these are team-roles-specific containers, not generic admonitions. Most content pages in how-to/, foundations/, and tutorials/ have no admonition markup at all.

Fix: (1) Document the canonical callout pattern in the UI Kit showcase (services/portal/ui-kit/pages/components/). (2) Add a callout example to every authoring template (doc-howto.html, doc-tutorial.html, doc-reference.html). (3) Add "check: does this page need a Note or Warning callout?" to the 12-point checklist (see §11).

Owner suggestion: docs / design.

Gap 7 — Hard-coded hex in SVG elements on practice pages

Severity: SEV-3. Type: token. Impact × Effort: Low-to-medium impact (dark mode rendering is broken for SVG visuals on ~15 pages), medium effort (per-page SVG rework).

Evidence:

  • team/roles/qa/practices/test-plan-authoring.html:60–83 — inline SVG uses style="--accent:#16a34a", --accent:#dc2626, --accent:#2563eb, --accent:#8b5cf6, --accent:#f59e0b on <rect> elements.
  • team/roles/sa/practices/capacity-finops.html:58–59 — style="stroke:#16a34a" and stroke:#0d9488 on <polyline>.
  • team/roles/qa/practices/shift-left-testing.html:64–80 — fill="#dc2626", fill="#16a34a", fill="#f59e0b" as SVG attributes.
  • handbook/sa/templates/screen-spec.html:125–130 — SVG annotation with fill="#0b1220", fill="#7dd3fc", fill="#fbbf24".

Fix: Replace raw hex with currentColor where the SVG element should inherit the text color, or with var(--success) / var(--danger) / var(--info) / var(--warn) for semantic fills.

Owner suggestion: docs (per-page SVG updates).

11. 12-point pre-commit checklist

Run this checklist in under 20 minutes before any change to services/frontend/portal/ or services/portal/internal/.

  1. Asset paths: all <link rel="stylesheet"> and <script> src attributes reference assets_v2/, not legacy assets/ paths (except assets/docs-syntax-theme.css imported via entry.css).
  2. Page type: data-page-type is present on <body> and uses a canonical value: reference, how-to, explanation, tutorial, landing, landing-section, cockpit, profile, or role-onboarding.
  3. Maintainer ID: data-maintainer-ids is present on <body>.
  4. Navigation source: data-nav-src on the <aside data-component="sidebar"> points to the correct nav-tree-{portal}.json. No other nav source is used.
  5. No inline styles: zero style="…" attributes on any element. If layout or sizing is needed, promote to a kit component class or utility class in the relevant component CSS file.
  6. No raw color literals: zero hex, rgb(), hsl() literals in page HTML (SVG fill/stroke included). Use currentColor or var(--token-name).
  7. Page history: a <footer class="docs-history"> block is present with at least one entry (exempt: redirect stubs, pdoc-generated code-reference pages).
  8. Status pills: all pill elements use only canonical tones — docs-pill--{success|info|warn|danger|neutral|rose|teal|accent}. No custom modifier classes (state-pill, raci-pill) without a corresponding kit definition.
  9. Tables: every <table> is wrapped in <div class="docs-table-wrap"> and the table element carries class="docs-table".
  10. Section index hero: new section index pages use <section class="home-hero home-hero--section"> — not docs-section-hero (which is the aspirational template class, not yet in use).
  11. Callout check: if the page contains caveats, prerequisites, important warnings, or tips — wrap them in <div class="docs-callout docs-callout--{note|warn|tip|danger}"> rather than plain paragraphs.
  12. make docs-check: run make docs-check and confirm zero errors before staging. This is the mandatory pre-commit gate (per MEMORY 2026-05-27).
