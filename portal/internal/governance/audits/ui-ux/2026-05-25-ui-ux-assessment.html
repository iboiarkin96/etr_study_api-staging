Overview

Assessment type: UI/UX consistency. Formation date: 2026-05-25.
Compare the portal UI to good practice (Table 1), score current state (Table 2), and track prioritized gaps (§ Scoring summary).

Scope and methodology

Scope

In scope: design tokens (color, type, spacing, motion), UI Kit v3 components, page templates per entity (article, hub, radar, practice, audit), status vocabulary, sidebar / topbar / breadcrumbs, mobile contract, theme toggle, and motion grammar. Out of scope: information architecture (covered by DX audit) and product UI outside the docs portal.

Methodology

Scores assigned by sampling 5 pages per entity type, inspecting assets_v2/ui-kit/tokens/ and component CSS, the page-template canonicalisation status, and a hand-walk of three Top-tier benchmark surfaces (Stripe Docs, Linear, Vercel). Each Table 2 row carries a justification. Final score is a weighted average across five axes.

Scoring summary

Narrative overall
