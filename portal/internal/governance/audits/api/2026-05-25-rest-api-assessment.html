Overview

Assessment type: REST API. Formation date: 2026-05-25.
Compare the HTTP surface to good practice (Table 1), score current state (Table 2), and track prioritized gaps (§ Scoring summary).

Scope and methodology

Scope

In scope: the OpenAPI 3.1 contracts under services/*/openapi/, the gateway handlers, error envelope, idempotency, rate-limit and security headers, metrics emission, and CI gates that protect them. Out of scope: gRPC internal traffic, browser-side rendering, the docs portal itself (covered by the DX and UI/UX assessments).

Methodology

Scores are assigned by inspecting the OpenAPI spec, gateway middleware, make api-validate output, Spectral / vacuum lint configuration, and the Prometheus scrape. Each Table 2 row carries a justification cell. Final score is the arithmetic mean of Table 2 rows.

Table 1: Reference practices

What "good" looks like for a public REST API. One row per practice.

# Practice Why it matters
1OpenAPI 3.1 contract checked into the repoSingle source of truth for clients, docs, and tests.
2Spec is linted in CI (Spectral / vacuum)Catches drift, missing descriptions, undocumented codes before merge.
3Examples for every request and response bodySDKs and docs render with realistic payloads; review burden drops.
4Stable error envelope (RFC 7807 problem+json)Clients can branch on machine-readable type/code, not free-text.
5Idempotency keys on all unsafe writesSafe retries under network partitions; no duplicated charges/orders.
6Cursor-based pagination with stable orderingPredictable scans; large lists do not break under inserts.
7Filtering, sorting, sparse fieldsets documentedAvoids ad-hoc query strings; matches JSON:API / Stripe norms.
8Resource-oriented URLs and HTTP verbsPredictable surface; caches, proxies, and tooling behave correctly.
9Versioning policy (URL or header) is documentedBreaking changes are explicit and discoverable.
10Deprecation policy with Sunset headerClients have a programmatic signal before endpoints disappear.
11Authentication uses OAuth 2.1 / OIDC bearer tokensIndustry default; integrates with existing identity providers.
12Authorization is scope/role-checked at the edge and the serviceDefense in depth; gateway misconfig cannot grant access.
13Per-key and per-IP rate limiting with RateLimit-* headersProtects shared resources; clients can back off cleanly.
14Security headers (HSTS, CSP, X-Content-Type-Options)Reduces blast radius of common browser-vector exploits.
15TLS-only with HSTS preload and modern ciphersEliminates downgrade attacks; required by app-store and SOC 2 audits.
16Structured request/response logging with trace IDsEnd-to-end debugging across services; correlatable to traces and metrics.
17RED metrics (rate / errors / duration) per routeStandard observability shape; supports SLOs and capacity planning.
18Distributed tracing on every external callLatency hotspots are visible without printf-debugging.
19Contract tests run on every PRSpec and implementation cannot drift silently.
20SDKs generated from the spec on tagConsumers always have a typed client; cuts integration time.
21Public changelog tied to spec versionsCustomers see what changed without diffing the spec.
22Webhooks documented with retry semantics and HMAC signingAsync surface is treated as first-class API, not an afterthought.
23Performance budget per endpoint (p50/p95 SLO)Regressions are caught in CI, not by paging customers.
24OWASP API Top-10 review in the release checklistCatches BOLA / mass assignment / SSRF before shipping.
