🧾 Governance · Traceability matrix

Traceability matrix — Study App today

The live audit of every commitment this service makes. Each row is one feature or cross-cutting guarantee Study App has promised; each column is the artefact that proves it: the team that owns it · the ADR that froze the decision · the API operation that implements it · the test that catches regressions · the runbook on-call opens when production breaks. A blank cell is a backlog item, not a styling gap.

This page is the reference — what is true right now, refreshed against the repo each docs-fix pass. The methodology behind it — what each column means, the YAML stanza shape, anti-patterns reviewers reject — lives at SA reference · Traceability matrix (methodology).

Legend

Two vocabularies share the matrix: row status pills on the right column, and cell symbols inside any evidence column when the artefact is missing or partial.

Symbol Meaning What it tells the reviewer
ok Closed chain Every upstream cell filled — owner, decision, code, test, runbook. The row's promise is whole today.
wip In flight At least one cell is partial or scheduled in the active sprint window. Dated commitment on record.
gap Open backlog item Blank cell with no committed sprint. A backlog ticket, not a build-time error.
∅ Missing artefact No code, no test, no runbook on disk. Each one is its own ticket against the row owner.
… Partial coverage The artefact exists but does not cover the whole requirement yet (e.g., test guards happy path only).
n/a No code surface Process, infra, or CI concern — legal, not a gap. Most cross-cutting NFR rows have n/a in the API column.

What this page is — and is not

This page is the live reference. It contains:

  • Today's rows, with today's owners, today's ∅ / … / status pills.
  • A coverage snapshot dated to the last make docs-fix pass.
  • Cross-links from every cell into the artefact it points at (ADR · operation spec · runbook).

What it deliberately does not contain:

  • The contract behind the matrix. What each column means, how a row stays accurate, the YAML stanza shape, anti-patterns reviewers reject — that's the methodology page at SA reference · Traceability matrix.
  • Procedure for adding a row. That's SA · authoring · Add an ADR (upstream decision) followed by the YAML edit + CI lint described on the methodology page.
