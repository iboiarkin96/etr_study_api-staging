monitoring · Dependencies

Monitoring is a pure consumer: it reads from the api service and exposes UIs to operators. There is no service downstream of monitoring at runtime.

Upstream

ComponentSurface consumedNotes
api:8000/metrics (Prometheus exposition), :8000/ready (Blackbox HTTP probe), NDJSON log file (Filebeat tail)The only upstream service. Contract authority lives in api · API reference.
Docker Engine / Docker ComposeContainer runtimeThe whole stack runs as Compose services. Image authority: ADR 0015.
Prometheus / Blackbox / Grafana / Filebeat / Elasticsearch / Kibana official imagesPulled from public registriesVersions pinned in Compose. Vulnerability scanning policy: ADR 0019 (extends to base images by analogy).

Downstream — who consumes monitoring

ConsumerSurfaceNotes
Operators (humans)Grafana UI · Kibana UI · Prometheus UILocal-only by default per ADR 0023; no external auth.
On-call routing (future)Webhook / paging integrationNot configured today — solo project. When added, page-routing rules live in Grafana alert rules and reference monitoring · On-call.
