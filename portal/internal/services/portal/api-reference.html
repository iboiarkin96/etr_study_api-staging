Reference Static URL tree No HTTP API

portal · API reference

The portal exposes no HTTP API. Its surface is a static URL tree served over HTTPS. "API reference" here means the contract a reader (or a deep-link) can rely on: top-level URLs, the search index format, and the asset versioning rules. Catalog spine slot kept for parity with the api / monitoring entries — see RFC 0004 §4.4.

URL surface — what a reader can deep-link

Search bundle — Pagefind contract

The portal ships a Pagefind search bundle at a fixed path. Per ADR 0033 (which supersedes ADR 0027), the file layout and search API are owned by Pagefind upstream; what we own is the pre-pass that tags pages and the runtime that calls pagefind.search(query).

  • Path: services/frontend/portal/pagefind/ — a chunked directory (pagefind.js, wasm.en.pagefind, fragment/*.pf_fragment, index/*.pf_index, filter/*.pf_filter, *.pf_meta). Lazily loaded by the topbar on first focus. Gitignored — rebuilt on every make docs-fix locally and on every CI Pages deploy.
  • Filters / facets: visibility = "public" | "internal"; kind = ADR · RFC · runbook · tutorial · how-to · reference · explanation · practice · landing · service. Both injected by the build-time pre-pass.
  • Producer: tools/docs/build_pagefind_index.py (step [10/10] of make docs-fix); CI: .github/workflows/deploy-prod.yml.
  • Visibility gate: tools/docs/check_pagefind_visibility.py — invoked right after the build; fails the run if any URL carries the wrong visibility tag.
  • KPI / SLO: per RFC 0002.

Asset versioning

Shared CSS / JS / fonts under services/frontend/portal/assets_v2/ are referenced by relative path from every HTML page. There is no fingerprinting at build time today; cache busts ride on edits to the asset filenames themselves (see ui-kit · Architecture).

If long-term caching becomes a problem, the right fix is to add a fingerprint suffix in the build step — not to add a runtime layer. Track in backlog.
