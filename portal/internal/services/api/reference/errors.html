Error catalog and sync

Our error management framework is built as a clear contract using stable identifiers that guarantee both consistency and predictability. Tables below are code-driven — patched by tools/docs/sync_docs.py from services/api/app/errors/*.py.

Source of truth: services/api/app/errors/common.py & services/api/app/errors/user.py. The rows between the <!-- BEGIN:… --> / <!-- END:… --> markers are regenerated on every make docs-fix run; CI fails the build via make docs-check when this page drifts from the Python source. Never edit those rows by hand.

Overview

Standardized error codes let external developers integrate with confidence, and let our automated monitoring, analytics, and triage stay precise across teams. A robust and transparent error system reduces support load, strengthens compliance, and enables better SLAs for partners and internal stakeholders.

Synchronization mechanism

  1. Define or update stable errors in services/api/app/errors/common.py and services/api/app/errors/user.py.
  2. Update OpenAPI examples in services/api/app/openapi/examples/errors.py and router responses.
  3. Run make docs-fix (calls tools/docs/sync_docs.py).
  4. The script patches marker blocks on this page, so catalog tables stay code-driven.
  5. Run make docs-check / make verify before merge.
