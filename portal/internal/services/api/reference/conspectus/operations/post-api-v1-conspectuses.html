Overview

Create a new conspectus for an identified learner. The body carries the Extract-and-Transform output (cue sheet + dense paragraph + bullets); the server persists the content, seeds an initial conspectus_schedules row at slot=A per the active SchedulePolicy, and emits a CREATED event.

§1Business context

The learner has just finished an Extract-and-Transform pass per the ETR methodology. They need to save that work as one durable artefact (the conspectus) so spaced retrieval can kick in starting one hour later (Slot A retrieve cadence).

This endpoint is the only way to create a conspectus row and its associated schedule. The two are created atomically — there is no half-state where a conspectus exists without a schedule row (per system-design · persistence).

§2User stories

  • US-CONSP-CREATE-1. As a learner, I want to save my Transform output as one note, so the system can remind me to retrieve it on a schedule.
  • US-CONSP-CREATE-2. As an integrator, I want creation to be idempotent so that retrying after a network blip never creates a duplicate note.
  • US-CONSP-CREATE-3. As a product owner, I want every new note to start at Slot A with a deterministic next_review_at so the due-list is predictable.

§6Authorization & permissions

API key per shared auth rules; learner resolved from (system_user_id, system_uuid) in the body. No additional scope. Ownership rule: the resolved learner becomes the owner_client_uuid on the new conspectuses row — no admin override, no proxy creation.

§10Side effects

  • On success. One transaction:
    1. INSERT INTO conspectuses (content snapshot, cue_sheet_schema_version=1, content_version=1).
    2. INSERT INTO conspectus_schedules (slot=A, slot_d_ladder_index=0, next_review_at=now()+PT1H, schedule_revision=1, FK to active policy).
    3. INSERT INTO conspectus_events (event_type="CREATED").
    4. INSERT INTO idempotency_keys with the response.
    5. Emit log event conspectus_created (info).
  • On idempotent replay. No DB writes; replay stored 201 body; emit conspectus_create_idempotent_replay (info).
  • On any error. Transaction rolls back; no partial state.

§11Idempotency & concurrency

  • Namespace: POST /api/v1/conspectuses.
  • Key composition: Idempotency-Key header + SHA-256 of canonical JSON body.
  • Replay status: 201 Created with stored body byte-for-byte.
  • TTL: 24 h (NFR-003).
  • Concurrency model: not applicable — fresh row, server-generated UUID prevents collisions.

§13Edge cases checklist

  • cue_sheet as top-level array. Reject — must be object (422).
  • cue_sheet with extra unknown keys. Accept (free-form object); shape recommendation lives on the resource hub.
  • bullets is empty array. Reject (422); minimum 1 bullet.
  • Repeated idempotent replays. All return the original 201; alert if replay rate is abnormally high.
  • SchedulePolicy missing in DB. Operator misconfiguration — 500 with conspectus_create_policy_missing at error level.
  • Unicode in paragraph/bullets. Accept; size check is on bytes (UTF-8).
  • Concurrent creates with different keys. Both succeed; each gets its own UUID and schedule row.

§14Observability requirements

Conventions: shared conventions.

Log events

  • conspectus_created (info) — {conspectus_uuid, owner_client_uuid, schedule_policy_id, algorithm_version}.
  • conspectus_create_idempotent_replay (info) — {conspectus_uuid, idempotency_key_hash}.
  • conspectus_create_user_not_found (info) — {requesting_system_user_id}.
  • conspectus_create_policy_missing (error) — invariant violation.

Metrics

  • conspectus_create_total (counter) — labels: status_code.
  • conspectus_create_duration_seconds (histogram).

Alerts

  • Page: conspectus_create_policy_missing any occurrence.
  • Page: 5xx rate > 1 % over 5 min.

  • Integration: AC-001 happy path; AC-002 replay; AC-003 user not found; AC-004 idempotency conflict; AC-006 atomic commit (verify both rows exist); AC-007 CREATED event.
  • Contract: Schemathesis on createConspectus — no 5xx on schema-conforming input.
  • End-to-end: create → list-due (within 1 h window) returns the new row.

  • Q1. Should the integrator be allowed to override schedule_policy_id on create? v1: no. Owner: product. Deadline: before 2026-06-01.
  • Q2. Hard caps on paragraph (4000 chars) and bullet count (20) — strict or warn? Current draft: enforce strictly. Owner: Ivan Boyarkin.

§18Out of scope

  • Bulk create (multiple conspectuses in one request).
  • Server-side validation of pedagogical quality (no NLP).
  • Image / attachment uploads — content is text-only in v1.
  • Webhook/event-bus emission — only in-DB events (conspectus_events).
