Field naming & format conventions

Single source of truth for field naming, types, formats, timezones, and PII classification across the internal API. Per-operation field tables in Section 7 / 8 of an analyst spec inherit these defaults; deviations must be documented in the spec with a reason.

Naming

  • JSON keys: snake_case. Always.
  • Boolean fields: is_* / has_* / can_* prefix (is_archived, has_due_items). Never inverted (is_not_active) — use the positive form.
  • Date / time fields: suffix _at for instants (created_at, next_review_at); suffix _on for calendar dates (due_on, when added).
  • Counts: suffix _count (review_count); never num_… or n_….
  • Identifiers: suffix _uuid for UUIDs (conspectus_uuid, client_uuid); suffix _id for non-UUID identifiers (system_user_id, schedule_policy_id).
  • Versions: algorithm_version (semver string), schedule_revision (monotonic integer), cue_sheet_schema_version (integer).
  • Avoid reserved-word collisions across stacks: do not use type, id alone, class, schema as bare field names.

Timezones

  • The wire format for instants is always UTC (Z-suffixed RFC 3339).
  • The user's preferred timezone is stored on users.timezone as an IANA string (e.g. Europe/Berlin) and used server-side to compute calendar-day boundaries (e.g. "due today").
  • If users.timezone is not set, calendar-boundary endpoints document UTC fallback explicitly in their spec.
  • Clients never receive a non-UTC instant. Local rendering is the client's responsibility.

Nullability and defaults

  • Use absence (omit the field) for "not provided" on requests; use null only when the API has to distinguish "explicitly cleared" from "unchanged" (rare).
  • On responses, prefer omitting null-valued optional fields. The OpenAPI says nullable: true only when the absence has its own meaning.
  • Defaults that the server applies on create live in the OpenAPI as default: on the request schema, not the response; the response always carries the resolved value.

Anti-patterns

  • camelCase JSON keys. Forbidden — we use snake_case everywhere on the wire and convert to language-native style in clients.
  • Local timezones on the wire. Forbidden — UTC only.
  • Mixed types for the same logical field. Once a field is integer, it stays integer (changing to string is a breaking change).
  • Magic strings for enums. Always declare a component $ref with an explicit enum array.
  • Free-form metadata objects without a schema version. Always pair with {field}_schema_version.
