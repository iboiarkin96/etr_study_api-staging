Shared API standards

Cross-cutting conventions that every internal API operation page should comply with — error envelope, error catalog, pagination, idempotency, auth, observability, field conventions, OpenAPI authoring, versioning, and the spec definition-of-done.

Standards

  • Auth — header, strategy, error mapping.
  • Error envelope — canonical error response shape.
  • Error catalog — codes, HTTP statuses, retryability.
  • Field conventions — naming, types, units.
  • Idempotency — header, scope, replay rules.
  • Pagination — cursor format and limits.
  • Observability conventions — logging, request IDs, metric names.
  • OpenAPI authoring guide — how operations are described in the spec.
  • Versioning — API version policy and deprecation.
  • Definition of Done — artefact-agnostic; spec checklist is one of the four families.
  • Spec template — starting structure for a new operation page.
