ui-kit · Architecture

The UI Kit is a build-time design system: it ships static HTML, CSS, and JS — no runtime application logic. The source lives under services/frontend/portal/assets_v2/ and bundles into per-portal entry files.

Runtime model

The UI Kit is build-time: there is no server, no API, no auth. It ships static HTML, CSS, and JS, and any consumer page references the bundled entry assets for its portal.

Source assets live under services/frontend/portal/assets_v2/ (raw token / component sources). They are bundled into services/frontend/portal/assets_v2/runtime/{internal,public,ui-kit}/entry.{css,js}.

Each portal — internal/, public/, and the standalone ui-kit/ showcase — loads the runtime relevant to its audience. The internal portal currently transitions pages off the legacy docs.css stack onto the new entry.css stack; the showcase uses the new stack directly.

Build pipeline

No bundler — the runtime entry is a single CSS file with ordered @import rules, plus a single JS entry that imports component modules and mounts them against document. Component CSS files are listed once in the entry; cascade conflicts are resolved by the import order (last @import wins).

Consumer pages include the entry pair directly: 

<link rel="stylesheet" href=".../assets_v2/runtime/internal/entry.css">
      <script type="module" src=".../assets_v2/runtime/internal/entry.js"></script>
